> 论文：*Attention Is Not What You Need: Grassmann Flows as an Attention-Free Alternative for Sequence Modeling*
> 作者：Chong Zhang
> 链接：https://arxiv.org/abs/2512.19428

## 写在前面

第一次看到标题差点划走。又是一个蹭"Attention Is All You Need"热度的？这类论文我见过太多了，改个标题卖个噱头，打开一看全是注水。

但这次不一样。Chong Zhang 确实拿出了一些有分量的东西。

他的核心观察很犀利：自注意力本质上是一种"张量提升"（tensor lifting），表达能力炸裂，但数学上几乎不透明。L×L 的注意力矩阵你盯着看半天也说不清里面到底发生了什么。他提出的替代方案用的是 Grassmann 流形，拿数学家们研究了一百多年的几何结构来替代那个又爱又恨的全连接矩阵。

这篇不是翻译。我从自己的理解出发讲几个事情：自注意力到底哪里让人不安、Grassmann 流形是个什么玩意儿、Plücker 坐标凭什么能做序列建模，以及最关键的，这套方案到底行不行。

## 一、自注意力的"皇帝新衣"？

### 1.1 先回顾一下：自注意力到底在干什么

Transformer 出现之前，处理序列主要靠 RNN 和 LSTM。第 t 个 token 的信息只能通过前一个隐藏状态"传来"，就像传话游戏，传到最后面目全非。

2017 年 Vaswani 他们提出了 Transformer。核心思路极简：干嘛一步步传？直接让每个 token 跟所有 token 同时对话不就完了。

数学上，输入序列 $X \in \mathbb{R}^{L \times d}$（L 是序列长度，d 是隐藏维度）：

$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

$QK^T$ 产出那个 L×L 的注意力矩阵。每个 $A_{ij}$ 表示第 i 个 token 对第 j 个 token 的关注度。拿这个权重矩阵去加权 $V$，得到新的表示。

### 1.2 看起来完美，但代价是什么？

好东西，但有一个硬伤：计算复杂度。

$QK^T$ 是 L×L 的，算它要 $O(L^2 d_k)$，再乘 $V$ 又是 $O(L^2 d_v)$。L 一大，L² 直接把你送走。

所以这几年"线性注意力"的变体层出不穷：Linformer、Performer、Linear Transformer……思路一致，想办法把 $QK^T$ 矩阵"压缩"或"近似"，避免显式算那个巨大的 L×L 矩阵。

但这些方案多少都在"打补丁"。框架还是自注意力的框架，只是在计算上做了近似。Chong Zhang 问了一个更根本的问题：我们真需要这个框架吗？

### 1.3 张量提升：一种新的解读视角

这是整篇论文我最喜欢的部分。作者没有简单地说"自注意力不好我换个方案"，而是从一个更高的视角重新解读了自注意力的本质。

标准多头注意力可以理解为一种**张量提升**操作。L 个隐藏向量 $h_1, h_2, \ldots, h_L \in \mathbb{R}^d$，对每一对 $(h_i, h_j)$ 通过 $Q, K, V$ 变换算出一个标量注意力权重，混合信息。

但换个视角：不做 softmax 归一化，直接看 $QK^T$ 本身。它本质上是一个从 $\mathbb{R}^{L \times d}$ 到 $\mathbb{R}^{L \times L}$ 的映射。一个"向量序列"被"提升"成了一个"向量对的矩阵"。

用论文的话说：这种张量提升把隐藏向量映射到成对交互的高维空间，然后用梯度下降去约束这个高维张量。表达能力极强，但**数学上不透明**（mathematically opaque）。多层叠加之后，你几乎没法用一组有限的显式不变量来描述模型在干什么。

我的理解：自注意力给了模型一台万能机器，它几乎可以做任何事（万能近似定理保证了这一点），但你完全不知道内部怎么运作的。这不就是深度学习被诟病的"黑箱"根源之一么。

## 二、Grassmann 流形：一种"有结构"的替代方案

### 2.1 为什么需要流形？

自注意力的问题是"太自由、太无结构"。那自然的方向就是找一个**有明确结构的数学对象**做替代。

流形（manifold）就是"局部像欧几里得空间，但全局有复杂拓扑结构"的东西。地球是经典例子：局部看地面是平的，全局是球面。

论文选的不是普通流形，而是 **Grassmann 流形**（Grassmann manifold）。代数几何和微分几何里的重要角色，线性代数的一种"升级版"。

### 2.2 Grassmann 流形的直观理解

$Gr(k, r)$ 是 $\mathbb{R}^r$ 中所有 k 维线性子空间的集合。

别被定义吓到。看例子。

**$Gr(1, 2)$**：$\mathbb{R}^2$ 中所有过原点的直线。每条直线由方向角 $\theta$ 确定，$\theta$ 和 $\theta + \pi$ 是同一条。站在原点往四面八方看，每条视线就是一个一维子空间。

**$Gr(2, 3)$**：$\mathbb{R}^3$ 中所有过原点的平面。每个平面由两个基向量张成。

**论文用的 $Gr(2, r)$**：$\mathbb{R}^r$ 中所有二维子空间的集合。嵌入在 r 维空间中的"小平面"。

### 2.3 为什么选 Grassmann 流形？

论文的逻辑链条是这样的：

1. 序列建模中，token 之间的交互本质上是**成对关系**
2. 自注意力用 L×L 矩阵表示成对关系，但空间太大（$L^2$ 维），不可解释
3. 用**二维子空间**表示"一对 token 之间的关系"，每对关系就对应 Grassmann 流形上的一个点
4. 流形维度固定，不随序列长度增长
5. 在流形上有黎曼度量、测地线、Plücker 嵌入等成熟的几何工具

打个不太严谨的比方：传统自注意力是让每个人跟所有人一对一见面，记一个好感分数（L×L 矩阵）。Grassmann 流形则是给每对人记录一组"关系特征"——共同兴趣、价值观匹配度之类的——编码成一个关系向量，所有可能的关系向量构成一个有限维的流形。

### 2.4 Grassmann 流形上的一些关键性质

在讲架构之前，补几个数学概念。

**维度：** $Gr(k, r)$ 的维度是 $k(r - k)$。论文用 $k = 2$，所以维度是 $2(r - 2) = 2r - 4$。当 $r = 32$ 时，$Gr(2, 32)$ 的维度是 60。

**旋转不变性：** 一个二维子空间可以用无数组不同的基向量来表示，但子空间本身不变。就像一个平面可以用无数种坐标系描述，平面还是那个平面。这对模型的"语义不变性"很关键。

**几何结构：** 有自然的黎曼度量，可以算两个子空间之间的距离。在一个有明确几何意义的框架下比较不同 token 对之间的关系。

## 三、Plücker 坐标：把子空间变成向量

### 3.1 核心问题：如何"编码"一个子空间？

有了 Grassmann 流形的概念，下一个问题：**怎么在计算机里表示一个二维子空间？**

最直接的想法是用两个基向量。但基的选择有无穷多种，$(u, v)$ 和 $(u + v, v)$ 张成的是同一个子空间。我们需要一种"规范化的表示"，唯一确定一个子空间。

**Plücker 坐标**就是干这个的。

### 3.2 Plücker 坐标的数学定义

给定 $u, v \in \mathbb{R}^r$，Plücker 坐标是一个 $\binom{r}{2}$ 维向量，每个分量是 $u$ 和 $v$ 对应分量的"行列式"：

$$p_{ij} = u_i v_j - u_j v_i, \quad \text{对所有 } 1 \leq i < j \leq r$$

$p = (p_{12}, p_{13}, \ldots, p_{r-1,r}) \in \mathbb{R}^{\binom{r}{2}}$ 就是这个子空间的 Plücker 坐标。你可以把它理解成子空间的"指纹"，归一化后唯一确定。

### 3.3 Plücker 坐标的关键性质

**不依赖基的选择。** 用 $(u', v')$ 张成同一个子空间（$u' = \alpha u + \beta v$，$v' = \gamma u + \delta v$，$\alpha\delta - \beta\gamma \neq 0$），新的 Plücker 坐标 $p' = (\alpha\delta - \beta\gamma) p$。只是缩放，方向没变。归一化后完全一致。

**维度有限且固定。** $r = 32$ 时，$\binom{32}{2} = 496$。每个子空间用 496 维向量表示，不随序列长度变化。

**Plücker 关系式。** 并非 $\mathbb{R}^{\binom{r}{2}}$ 中的所有向量都是合法的 Plücker 坐标。合法坐标必须满足一组二次方程，定义了 Plücker 嵌入像——Grassmann 流形在高维空间中的表示。

### 3.4 一个具体的例子

$r = 3$，$\binom{3}{2} = 3$。给定两个向量：

$$u = (u_1, u_2, u_3), \quad v = (v_1, v_2, v_3)$$

Plücker 坐标：

$$p_1 = p_{12} = u_1 v_2 - u_2 v_1$$
$$p_2 = p_{13} = u_1 v_3 - u_3 u_1$$
$$p_3 = p_{23} = u_2 v_3 - u_3 v_2$$

注意：这个向量 $(p_1, p_2, p_3)$ 恰好是 $u$ 和 $v$ 的**叉积**（cross product）！在三维空间中，叉积方向垂直于两向量张成的平面，长度等于平行四边形面积。所以 Plücker 坐标编码了子空间的"方向"和"大小"。

更高维的情况，Plücker 坐标就是外积（wedge product）的分量。

### 3.5 我的思考：为什么 Plücker 坐标特别适合 token 对？

我觉得这个选择很巧妙。Plücker 坐标本质上是两个向量的外积分量，抓住的是两个向量的"相对位置"和"张成的面积"，而不是各自的绝对信息。

语义层面，这不正是我们想要的？分析"国王"和"王后"之间的关系时，我们在乎的是它们之间的"相对语义"，不是"国王"这个概念本身包含什么。

具体来说：
- 两个 token 的表示几乎平行（语义相似），Plücker 坐标模长很小
- 表示正交（语义无关），模长较大
- 表示反平行（语义对立），模长也较大

"只看相对关系，不看绝对位置"，自注意力的 $Q_i \cdot K_j$ 也在做类似的事。但 Plücker 坐标提供了更有结构的表示。

## 四、因果 Grassmann 层：架构详解

数学铺垫够了，看实际架构。论文的核心模块叫 **因果 Grassmann 层**（Causal Grassmann Layer），替换 Transformer 中的多头注意力层。

### 4.1 整体架构一览

```
输入 h_t ∈ ℝ^d
    ↓
(1) 线性降维: z_t = W_red · h_t ∈ ℝ^r
    ↓
(2) 多尺度局部配对: 对每个偏移 Δ ∈ W, 形成 (z_t, z_{t+Δ})
    ↓
(3) Plücker 编码: 计算每对的 Plücker 坐标, 归一化
    ↓
(4) 投影回模型空间: g_t = W_plü · [各偏移的 Plücker 特征]
    ↓
(5) 门控融合: h_t' = gate · g_t + (1 - gate) · h_t
    ↓
(6) FFN + 残差 + LayerNorm (标准后处理)
```

### 4.2 第一步：线性降维

$$z_t = W_{\text{red}} h_t + b_{\text{red}}, \quad W_{\text{red}} \in \mathbb{R}^{r \times d}$$

把隐藏状态从 $\mathbb{R}^d$ 投影到 $\mathbb{R}^r$，$r \ll d$。论文中 $d = 256$，$r = 32$。

两个原因。Plücker 坐标的维度是 $\binom{r}{2}$，$r = d = 256$ 时 $\binom{256}{2} = 32640$ 维，太离谱了。降到 $r = 32$ 后只有 496 维，可控。降维矩阵 $W_{\text{red}}$ 在训练中学习，自动选择对下游 Grassmann 交互最重要的信息，相当于一个信息瓶颈。

### 4.3 第二步：多尺度局部配对

因果 Grassmann 层的关键创新。

给定 $z_t \in \mathbb{R}^r$，对每个位置 $t$ 和每个偏移量 $\Delta \in \mathcal{W}$，形成 $(z_t, z_{t+\Delta})$ 对。

论文用的偏移窗口：

$$\mathcal{W} = \{1, 2, 4, 8, 12, 16\}$$

每个 token 跟后面 1、2、4、8、12、16 个位置配对。多尺度是为了捕获不同距离的依赖：$\Delta = 1, 2$ 管局部 bigram/trigram 信息，$\Delta = 4, 8$ 跨短语和从句，$\Delta = 12, 16$ 捕获更远的信息。

跟计算机视觉里的多尺度特征提取一个道理：ResNet 在不同分辨率上提取特征再融合。不同尺度的信息有不同的价值。

因果性要注意：配对是"向后看"的，$z_{t+\Delta}$ 只出现在 $z_t$ 后面。保证当前 token 只依赖过去信息，不能偷看未来。

### 4.4 第三步：Plücker 编码

对每一对 $(z_t, z_{t+\Delta})$，计算 Plücker 坐标：

$$p_t^{(\Delta)} = z_t \wedge z_{t+\Delta}$$

分量形式：

$$p_{t,ij}^{(\Delta)} = z_{t,i} \cdot z_{t+\Delta,j} - z_{t,j} \cdot z_{t+\Delta,i}, \quad \forall 1 \leq i < j \leq r$$

归一化：

$$\hat{p}_t^{(\Delta)} = \frac{p_t^{(\Delta)}}{\|p_t^{(\Delta)}\|_2}$$

归一化确保只编码"子空间的方向"（相对关系），不编码"大小"（绝对幅度）。

计算复杂度：对每个位置 $t$ 和偏移 $\Delta$ 计算 $\binom{r}{2}$ 个分量。总共 $L \times |\mathcal{W}|$ 个对，所以是 $O(L \cdot |\mathcal{W}| \cdot r^2)$。$r$ 和 $|\mathcal{W}|$ 固定时，关于 L 线性。

### 4.5 第四步：投影回模型空间

归一化后的 Plücker 特征 $\hat{p}_t^{(\Delta)}$ 每个 $\binom{r}{2}$ 维，需要映射回隐藏维度 $d$。

把所有偏移的特征拼接，用线性层投影：

$$\tilde{g}_t = W_{\text{plü}} \cdot \text{concat}\left(\hat{p}_t^{(\Delta_1)}, \hat{p}_t^{(\Delta_2)}, \ldots, \hat{p}_t^{(\Delta_m)}\right)$$

$W_{\text{plü}} \in \mathbb{R}^{d \times m \cdot \binom{r}{2}}$，$m = |\mathcal{W}|$。

$r = 32$，$m = 6$ 时，拼接后 $6 \times 496 = 2976$ 维，然后映射到 $d = 256$ 维。这步就是"翻译官"，把 Grassmann 流形上的信息翻译回 $\mathbb{R}^d$ 空间跟后续网络层对接。

### 4.6 第五步：门控融合

把几何特征融合回原始隐藏状态：

$$\text{gate}_t = \sigma\left(W_{\text{gate}} \cdot [h_t; g_t] + b_{\text{gate}}\right)$$

$$h_t' = \text{gate}_t \odot g_t + (1 - \text{gate}_t) \odot h_t$$

$\sigma$ 是 sigmoid，$[h_t; g_t]$ 是拼接，$\odot$ 是逐元素乘法。

每个维度独立决定"保留多少原始信息"和"注入多少几何信息"。gate 接近 1 就主要用几何特征，接近 0 就主要保留原始状态。

不是所有 token 都需要"更新"自己的理解。"苹果"在大多数语境下就是那个水果，含义明确；"bank" 到底是银行还是河岸，得看上下文。门控机制让模型灵活选择何时听取 Grassmann 交互的信息，何时保持自己的判断。

### 4.7 第六步：标准后处理

门控融合之后，接 FFN、残差连接、层归一化：

$$\text{output}_t = \text{LayerNorm}\left(h_t' + \text{FFN}(h_t')\right)$$

$$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$

标准 Transformer 后处理，没什么特别的。

## 五、完整模型架构：GrassmannLM

### 5.1 把因果 Grassmann 层堆起来

GrassmannLM 的结构跟 Transformer 很像：

1. **Token 嵌入层**：词嵌入 + 位置编码
2. **N 个因果 Grassmann 层**：每层包含完整流程（降维 → 配对 → Plücker 编码 → 投影 → 门控融合 → FFN）
3. **输出层**：线性层 + softmax，预测下一个 token

论文测了两种配置：6 层（约 1300 万参数）和 12 层（约 1800 万参数）。

### 5.2 跟 Transformer 的完整对比

| 特性 | Transformer | GrassmannLM |
|------|------------|-------------|
| 核心交互机制 | 自注意力（$QK^T$ 矩阵） | Grassmann 流形上的子空间交互 |
| 序列间信息传递 | 通过 L×L 注意力矩阵 | 通过多尺度局部配对 + Plücker 编码 |
| 空间结构 | 无结构（平坦的 $\mathbb{R}^{L \times L}$ 空间） | 有结构（Grassmann 流形，有几何意义） |
| 计算复杂度 | $O(L^2d + Ld^2)$ | $O(Ld^2)$（关于 L 线性） |
| 内存占用 | $O(L^2)$（需要存储注意力矩阵） | $O(L)$ |
| 可解释性 | 低（注意力权重提供一些线索，整体不透明） | 高（Plücker 坐标有明确几何意义） |
| 数学基础 | 线性代数 + softmax | 代数几何 + 微分几何 |
| 全局长程依赖 | 天然支持 | 通过堆叠多层逐步传播 |

### 5.3 复杂度分析

拆开看因果 Grassmann 层的计算量：

1. **线性降维：** $W_{\text{red}} \in \mathbb{R}^{r \times d}$，$O(Lrd)$
2. **Plücker 编码：** 每对算 $\binom{r}{2}$ 个分量，$O(L \cdot m \cdot r^2 / 2)$
3. **投影回模型空间：** $W_{\text{plü}} \in \mathbb{R}^{d \times m \cdot \binom{r}{2}}$，$O(L \cdot d \cdot m \cdot r^2 / 2)$
4. **门控融合：** $W_{\text{gate}} \in \mathbb{R}^{d \times 2d}$，$O(L \cdot 2d^2)$
5. **FFN：** $O(L \cdot d \cdot d_{\text{ff}})$

注意到没有？**没有一项包含 $L^2$。** 全部关于 L 线性。因为我们从未构造 L×L 矩阵，每个 token 只跟固定数量的"邻居"交互。

但是。论文也很诚实地指出，当前 PyTorch 实现中 Plücker 坐标用循环逐对计算行列式，效率不高。而 GPU 上的自注意力（FlashAttention 等）经过大量优化，跑得很快。所以理论上 GrassmannLM 是线性的，实际运行速度可能反而比优化过的 Transformer 慢。

没吹牛。这是很诚实的评估。

## 六、实验结果：现实检验

### 6.1 Wikitext-2 语言建模

语言建模的标准考题。根据前文预测下一个词，困惑度越低越好。

| 模型 | 层数 | 参数量 | 验证困惑度 |
|------|------|--------|-----------|
| TransformerLM | 6 | ~13M | 241.0 - 253.6 |
| **GrassmannLM** | 6 | ~13M | **275.7** |
| TransformerLM | 12 | ~18M | 235.2 |
| **GrassmannLM** | 12 | ~18M | **261.1** |

坦白说，数字不太好看。同参数量下 GrassmannLM 的困惑度高了大约 10-15%。

不过我觉得有几点值得注意。

这是个全新的架构。Transformer 经过多年打磨，预归一化、GELU、更好的初始化等技巧都很成熟了。GrassmannLM 第一次尝试，超参数调优空间还很大。

差距在缩小。从 6 层到 12 层，Transformer 困惑度从 ~247 降到 235（提升约 5%），GrassmannLM 从 276 降到 261（提升约 5.4%）。深度增加对 GrassmannLM 帮助更大，说明架构本身有深度可扩展的潜力。

最重要的是：完全不使用自注意力就达到了这个水平。这本身就说明自注意力不是强性能的必要条件。

### 6.2 SNLI 自然语言推理

NLI 任务：给一个前提（premise）和一个假设（hypothesis），判断蕴含、矛盾还是中立。

论文的实验设置很聪明：**不替换整个模型**，只替换"头部"。用预训练好的 DistilBERT 做骨干网络，然后分别用 Transformer 头和 Grassmann-Plücker 头做分类。

| 头部 | 验证准确率 | 测试准确率 |
|------|-----------|-----------|
| Transformer 头 | 0.8545 | 0.8511 |
| **Grassmann-Plücker 头** | **0.8550** | **0.8538** |

这个结果很有意思。Grassmann 头**略优**。

我猜原因是 NLI 任务本质上就是在判断两个句子之间的"关系"，恰好就是 Plücker 坐标擅长的东西。两个句子的表示在 Grassmann 流形上的距离和方向，自然编码了它们的关系。自注意力虽然通用性更强，但在这种"关系判断"任务上可能反而杀鸡用牛刀了。

### 6.3 我的解读

综合两个实验，我的判断：GrassmannLM 目前还不能取代 Transformer，但展示了一个有价值的研究方向。

语言建模需要全局信息，只做局部配对的话信息得通过多层逐步传播，浅层模型覆盖不了足够长的距离，性能有差距。NLI 这种关系判断任务上，Grassmann 几何反而有独特优势。线性复杂度在长序列场景是实打实的利好。

让我想起 Mamba 和 SSM 的轨迹。最初的 SSM 性能也不如 Transformer，架构改进后（选择性 SSM、Mamba）差距逐渐缩小，特定场景下还反超了。GrassmannLM 可能也在类似的轨道上。

## 七、深度讨论：从"张量提升"到"流形演化"

### 7.1 自注意力的"不可解释性"到底意味着什么？

数学中，一个系统"可解释"通常意味着：你可以用一组有限的不变量来描述它的状态和行为。刚体运动可以用位置、速度、角速度等有限个量完全描述。

自注意力中，信息在 L×L 的矩阵里流动，多层变换之后，你几乎没法用任何有限的不变量族来"总结"模型学到了什么。每层都在做无结构的张量变换。

这不仅仅是"理解模型"的问题。它也影响泛化、鲁棒性和可控性。不可解释的模型，你很难预测它在分布外数据上怎么表现。

### 7.2 Grassmann 流形怎么解决这个问题？

Grassmann 流形提供了一个"有结构"的信息交互空间：维度有限（$Gr(2, r)$ 的维度是 $2r - 4$），每个点代表一个二维子空间有明确数学含义，有不变量（子空间维数、行列式、Plücker 关系式），有度量（可以算子空间之间的距离）。

在这种框架下，信息传播可以被理解为"语义流形上的几何演化"。每处理一个 token，语义子空间就在流形上做一次形变（deformation），形变累积起来形成模型对整个序列的理解。

### 7.3 通用近似定理还成立吗？

把信息交互限制在 Grassmann 流形上，表达能力会下降吗？

理论上不会。通用近似定理适用于任何足够"丰富"的空间。多尺度配对（6 种偏移）、多层堆叠（6-12 层）、非线性门控 + FFN，模型有足够的表达能力近似任意函数。

关键区别在于：Transformer 的近似发生在一个没有明确结构的高维空间里；GrassmannLM 的近似发生在一个有明确几何结构的有限维流形上。后者你可以更有信心地分析、理解和控制近似过程。

一个不太精确的比方：两者都能到目的地，一个是坐没有导航的热气球飘过去（可能飘得很快但你不知道经过了哪里），一个是沿有地图标记的公路开过去（路线有结构，你能知道经过了哪里）。

## 八、局部配对 vs 全局注意力：信息传播的"接力赛"

### 8.1 局部配对怎么捕获长距离依赖？

自注意力最强的能力之一是直接捕获任意距离的依赖。第 1 个词可以直接"看到"第 1000 个词。GrassmannLM 只用了局部配对（最大偏移 16），长距离依赖怎么办？

通过多层堆叠，信息像接力赛一样逐层传播。

12 层模型，每层最大偏移 16：
- 第 1 层：覆盖 16 步以内
- 第 2 层：间接覆盖约 32 步
- 第 12 层：理论上覆盖 $12 \times 16 = 192$ 步

这是理论上限。实际效率取决于门控机制是否有效传递远距离信息、多个偏移尺度的组合是否覆盖了足够的距离、训练中学到的信息传递策略。

### 8.2 跟卷积和循环的对比

这种设计让我想到两个经典方法。

**CNN**：局部感受野 + 层间堆叠扩大覆盖范围。GrassmannLM 的多尺度配对类似膨胀卷积（dilated convolution），用不同"步长"快速扩大感受野。

**RNN**：逐步传递信息覆盖整个序列。但 RNN 是串行的，必须先处理完 t 才能处理 t+1。GrassmannLM 的因果层可以并行处理所有位置。

GrassmannLM 结合了两者的优点：像 CNN 一样可并行，像膨胀卷积一样用多尺度快速扩大覆盖范围，但不局限于固定卷积核大小。

### 8.3 为什么不用更大的偏移？

$\mathcal{W} = \{1, 2, 4, 8, 12, 16\}$ 的最大值才 16，用 32、64、128 不就能更快覆盖长距离了？

可能的几个原因。更多偏移意味着更多 Plücker 坐标计算，虽然仍关于 L 线性但常数因子会增大。Wikitext-2 只有约 200 万训练 token，参数太多容易过拟合。距离太远的 token 关系噪声大，直接编码可能反而引入干扰。

但我觉得更大的偏移集和更多层数应该是提升性能的一个方向。论文也提到了这一点作为未来工作。

## 九、实现细节：如果你要自己复现

### 9.1 Plücker 坐标的计算实现

核心计算是外积。对 $u, v \in \mathbb{R}^r$，计算所有 $u_i v_j - u_j v_i$（$i < j$）。

```python
def plucker_coords(u, v):
    """
    u, v: (..., r)
    returns: (..., C(r,2))
    """
    # 利用外积的不对称部分
    outer = u[..., :, None] * v[..., None, :]  # (..., r, r)
    anti_sym = outer - outer.transpose(-1, -2)  # 取反对称部分
    # 取上三角部分（i < j）
    indices = torch.triu_indices(r, r, offset=1)
    return anti_sym[..., indices[0], indices[1]]
```

$r = 32$ 时输出维度 496。实现用了矩阵乘法和索引操作，GPU 上可以批量化处理。但论文指出优化空间还很大，特别是可以借鉴 FlashAttention 的思路减少内存访问。

### 9.2 完整的因果 Grassmann 层伪代码

```python
class CausalGrassmannLayer(nn.Module):
    def __init__(self, d, r, windows, d_ff):
        super().__init__()
        self.r = r
        self.windows = windows  # e.g., [1, 2, 4, 8, 12, 16]
        
        # (1) 线性降维
        self.W_red = nn.Linear(d, r)
        
        # (4) 投影回模型空间
        pluecker_dim = r * (r - 1) // 2
        self.W_plu = nn.Linear(len(windows) * pluecker_dim, d)
        
        # (5) 门控融合
        self.W_gate = nn.Linear(2 * d, d)
        
        # (6) FFN
        self.ffn = nn.Sequential(
            nn.Linear(d, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d)
        )
        self.norm1 = nn.LayerNorm(d)
        self.norm2 = nn.LayerNorm(d)
    
    def forward(self, h):
        # h: (L, d)
        L = h.size(0)
        
        # (1) 线性降维
        z = self.W_red(h)  # (L, r)
        
        # (2) + (3) 多尺度配对 + Plücker 编码
        pluecker_features = []
        for delta in self.windows:
            # 形成配对 (z_t, z_{t+delta})，注意因果性
            z_current = z[:-delta]   # (L-delta, r)
            z_future = z[delta:]     # (L-delta, r)
            
            # 计算 Plücker 坐标
            p = plucker_coords(z_current, z_future)  # (L-delta, C(r,2))
            p = F.normalize(p, dim=-1)  # 归一化
            
            # Pad 回 L 的长度（前面补零）
            p_padded = F.pad(p, (0, 0, delta, 0))  # (L, C(r,2))
            pluecker_features.append(p_padded)
        
        # 拼接所有偏移的特征
        all_features = torch.cat(pluecker_features, dim=-1)  # (L, m*C(r,2))
        
        # (4) 投影回模型空间
        g = self.W_plu(all_features)  # (L, d)
        
        # (5) 门控融合
        gate = torch.sigmoid(self.W_gate(torch.cat([h, g], dim=-1)))
        h_prime = gate * g + (1 - gate) * h
        
        # 残差 + LayerNorm
        h_prime = self.norm1(h + h_prime)
        
        # (6) FFN + 残差 + LayerNorm
        out = self.norm2(h_prime + self.ffn(h_prime))
        
        return out
```

### 9.3 关键超参数

| 参数 | 值 | 说明 |
|------|---|------|
| d（隐藏维度） | 256 | 模型的主要隐藏维度 |
| r（降维后维度） | 32 | Plücker 编码的工作空间维度 |
| $\mathcal{W}$（偏移窗口） | {1,2,4,8,12,16} | 多尺度局部配对的偏移量 |
| d_ff（FFN 维度） | 1024 | 前馈网络的中间维度 |
| 层数 | 6 或 12 | 因果 Grassmann 层的数量 |
| 学习率 | 5e-4 | AdamW 优化器 |
| 权重衰减 | 0.01 | AdamW 的 L2 正则化 |

## 十、Grassmann 流形的进一步思考

### 10.1 为什么选 k=2？

论文用 $Gr(2, r)$，即二维子空间。为什么不用更高的 $k$？

计算量。$Gr(k, r)$ 的 Plücker 坐标维度是 $\binom{r}{k}$，随 k 增长极快。$r = 32$ 时：$k = 2$ 对应 496 维，$k = 3$ 直接跳到 4960，$k = 4$ 到 35960。

二阶交互已经能捕获大部分有用信息。三阶及更高阶的交互边际收益递减。而且 $k = 2$ 时 Plücker 坐标有特别优雅的形式（类似叉积），几何直觉最清晰。

探索 $k = 3$ 或混合不同 k 值的架构是一个值得研究的方向。

### 10.2 Grassmann 流形上的"流"（Flows）

论文标题里的"Grassmann Flows"值得聊几句。微分几何中，流（flow）指连续的、随时间演化的变换。Grassmann 流形的语境下，信息处理可以理解为：随着序列位置从左到右推进，语义子空间在流形上连续"流动"和"变形"。

跟 RNN 的"隐藏状态演化"类似，但关键区别在于 RNN 的隐藏状态在 $\mathbb{R}^d$（平坦空间）中演化，GrassmannLM 的子空间在 $Gr(2, r)$（弯曲流形）中演化。弯曲流形上的演化有更丰富的结构：曲率、测地线、平行输运等，为后续研究提供了工具箱。

### 10.3 跟其他无注意力架构的关系

近几年无注意力架构不少：Mamba/SSM 用状态空间模型替代自注意力，RWKV 结合 RNN 和线性注意力，MLP-Mixer 纯靠 MLP，Hyena 用长卷积。

GrassmannLM 的区别在于**数学基础**。Mamba 基于线性系统理论，RWKV 基于线性注意力，MLP-Mixer 靠万能近似。GrassmannLM 基于代数几何，信息交互发生在有明确几何结构的流形上。这意味着它天然有更丰富的数学分析工具：微分几何分析信息流动，代数几何分析子空间性质和变换。

### 10.4 潜在的研究方向

几个我觉得有前景的方向：

**局部不变量。** 当前只用了 Plücker 坐标（外积），还可以加入行列式、内积、角度等几何不变量。

**自适应偏移。** 偏移集合目前是固定的，可以让模型根据输入动态选择（类似 Mamba 的选择性机制）。

**全局不变量。** 论文明确提到的未来方向。在局部 Plücker 特征上构建全局池化或聚合，实现对整个序列的全局理解。

**混合架构。** Grassmann 层和自注意力层混合使用，Grassmann 管局部，少量注意力层管全局依赖。

**硬件优化。** 开发针对 Plücker 坐标计算的高效 GPU kernel，缩小理论和实际性能之间的差距。

## 十一、我的思考与评价

### 11.1 这篇论文的价值在哪里？

最大价值不在"GrassmannLM 跑赢了 Transformer"，事实上还没跑赢。真正有价值的是它提供了一个全新的视角：序列理解本质上是一种几何演化，token 之间的关系可以编码为几何对象，信息传播可以理解为流形上的变形。

这种视角的好处：有成熟的数学工具（黎曼度量、测地线、曲率）可以用来分析；每个计算步骤有明确几何含义；不同的几何结构可以灵活组合。

### 11.2 为什么我觉得这个方向值得关注？

过去几年深度学习的发展很大程度上是工程驱动的：更好的优化器、更大的模型、更多的数据、更快的硬件。理论理解一直在追着实践跑。

Grassmann 流形的方法代表了一种"理论驱动"的尝试。不是先堆实验再事后解释，而是从数学原理出发设计有结构的架构。短期可能不如工程优化有效，但长期看，对"让深度学习变得可理解、可控、可信赖"可能更有帮助。

### 11.3 它的局限性

性能差距是真实的。Wikitext-2 上 10-15% 的差距说明当前架构信息传播效率还不够。

理论优势还没充分转化为实践优势。线性复杂度理论上很好，实际跑得反而慢。

缺乏大规模实验。两个基准测试都是小规模任务。更大规模的数据和模型上效果如何，不知道。

偏移集合 $\{1, 2, 4, 8, 12, 16\}$ 怎么选的？有没有更优的？论文没给出充分的理论分析。

### 11.4 更大的图景

这篇论文属于一个正在兴起的研究趋势：**用结构化的数学对象替代无结构的神经网络组件。**

类似的：Lie 群在旋转等变网络中的应用，图神经网络中的拓扑结构，流形正则化在表示学习中的应用，辛几何在 Hamiltonian 神经网络中的应用。共同点是不盲目增加容量，而是利用数学结构提高效率和可解释性。

GrassmannLM 就是这个趋势在序列建模领域的一个实例。

## 十二、动手指南：如果你想尝试

### 12.1 复现的步骤

建议按这个顺序来：

1. 先实现 Plücker 坐标计算函数，确保理解了核心操作
2. 实现单个因果 Grassmann 层，随机数据上验证前向/反向传播
3. 用 Grassmann 层替换 Transformer 的注意力层，PTB 之类的小数据集上快速验证
4. Wikitext-2 上训练完整模型，对比 Transformer baseline
5. SNLI 上试 Grassmann 头部，验证论文的 NLI 结果

### 12.2 关键注意事项

归一化很重要。Plücker 坐标计算后必须归一化，否则子空间大小会干扰学习。

因果 padding。序列起始位置后面的偏移对会"超出"序列长度，得正确处理。

学习率可能比 Transformer 更敏感，需要仔细调。Plücker 坐标涉及行列式计算，梯度消失/爆炸的风险要注意。

### 12.3 扩展实验建议

几个可以试的方向：不同偏移集合（纯指数增长 $\{1, 2, 4, 8, 16, 32\}$ 或等差数列 $\{1, 3, 5, 7, 9, 11\}$）、Grassmann 层之间穿插少量注意力层的混合架构、在 WikiText-103 或 C4 上验证更大规模数据效果、消融实验（去掉某个偏移、改降维维度 r、去掉门控机制等）。

## 十三、总结对比

### 13.1 核心差异一览

| 维度 | Transformer | GrassmannLM |
|------|------------|-------------|
| **核心思想** | 每个 token 直接跟所有 token 对话 | 用流形上的子空间编码局部关系 |
| **数学基础** | 线性代数 + softmax | 代数几何（Grassmann 流形 + Plücker 坐标） |
| **信息交互空间** | $\mathbb{R}^{L \times L}$（无结构） | $Gr(2, r)$（有限维弯曲流形） |
| **序列复杂度** | $O(L^2)$ | $O(L)$ |
| **内存** | $O(L^2)$ | $O(L)$ |
| **长距离依赖** | 一步到位 | 多层传播 |
| **可解释性** | 较低 | 较高（有几何意义） |
| **当前性能** | 更好 | 略逊（10-15% 差距） |
| **实现优化** | 高（FlashAttention 等） | 低 |
| **理论分析工具** | 较少 | 丰富（微分几何、代数几何） |

### 13.2 适用场景建议

| 场景 | 推荐架构 | 原因 |
|------|---------|------|
| 短序列 + 关系判断 | GrassmannLM | Plücker 坐标天然适合编码成对关系 |
| 超长序列（>10K tokens） | GrassmannLM（潜力） | 线性复杂度优势明显 |
| 通用语言建模 | Transformer（目前） | 当前性能更优 |
| 需要可解释性的应用 | GrassmannLM | 有几何解释 |
| 受限计算环境 | GrassmannLM（优化后） | 线性复杂度 + 更低内存 |

## 写在最后

读完这篇论文，最大的感受是：深度学习研究正在从"大力出奇迹"转向"巧劲出奇迹"。以前大家都在堆参数堆数据堆算力，看谁把 Transformer 做得更大。现在越来越多的人开始想，有没有更聪明的办法。

Grassmann 流形给了一个启发：也许不需要给模型一个完全自由的无结构巨大空间，而是给一个有引导的、有明确结构的较小空间。有地图和指南针，比无限大的空旷场地更有效率。

论文的不足是明显的。性能差距、缺乏大规模验证、实现效率低。但不影响它作为一个有价值的研究起点。

最有价值的不是 GrassmannLM 这个具体模型，而是它代表的思维方式：用结构化的数学对象来设计神经网络。这种思维方式未来可能催生更多有趣的架构。

至于"Attention Is All You Need"——更准确的版本大概是：Attention Is Enough, But Not All You Need.
