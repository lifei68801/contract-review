## Context-Compression VirusTotal 标记修复（20:40 北京时间）

### 问题
飞哥发现 context-compression 被 VirusTotal 标记为 "Suspicious"

### 原因
`gateway-compact.sh` 脚本包含可疑模式：
```bash
curl -s -X POST "http://localhost:$GATEWAY_PORT/api/session/compact" \
    -H "Authorization: Bearer $token" \
    -H "Content-Type: application/json" \
    -d "..."
```
VirusTotal 将 `curl + POST + Authorization: Bearer` 模式标记为可疑，即使目标是 localhost

### 修复
1. 删除了可疑脚本：
   - `gateway-compact.sh`（curl POST 模式）
   - `truncate-sessions-safe.sh.bak`（备份文件）
   - `truncate-sessions.sh`（旧版本）

2. 更新 SKILL.md：
   - 版本升级到 3.8.3
   - 删除所有对 gateway-compact.sh 的引用
   - 更新安全声明：零网络请求

### 待解决

- 需要等待 ClawHub 修复 API

## Context-Compression v3.9.2 活跃会话事实提取修复（23:50 北京时间）

### 问题
活跃会话（有 .lock 文件）被完全跳过，事实提取从未执行：
```
日志显示:
⏭️  Skip active: xxx.jsonl
Truncated: 0
Facts extracted: 0  ← 永远是 0
```

### 原因
`skipActive=true` 时，活跃会话直接 `continue`，不做任何处理。

### 修复
修改 `truncate-sessions-safe.sh`：
- 即使活跃会话跳过截断，也检查是否超过阈值
- 超过阈值时，扫描高优先级内容并提取事实
- 提取完成后再跳过

### 版本
- truncate-sessions-safe.sh: v9
- SKILL.md: v3.9.2

### 效果
- 长期活跃会话不会丢失重要记忆
- `Facts extracted` 将从 0 变为实际数量
