# 会话管理方案 v3 - 精细化控制

## 需求
1. 触发条件：上下文 > 100K 或 会话 > 10 轮
2. 上下文结构：最近 10 轮完整 + 历史压缩摘要
3. 存储位置：
   - 完整会话 → `~/.openclaw/agents/main/sessions/*.jsonl`
   - 压缩摘要 → `memory/sessions/*.md`
4. 目标：不超限、连贯性、记住历史

---

## 递归压缩机制

### 问题
当"最近10轮 + 历史压缩"仍超过100K时，如何处理？

### 解决方案：分层压缩

```
┌─────────────────────────────────────┐
│ 第1层：最近 10 轮完整（~30K）         │
├─────────────────────────────────────┤
│ 第2层：当日摘要（~10K）               │
├─────────────────────────────────────┤
│ 第3层：本周摘要（~5K）                │
├─────────────────────────────────────┤
│ 第4层：长期记忆（~5K）                │
└─────────────────────────────────────┘
     总计 ≤ 50K（安全余量）
```

### 压缩策略

| 层级 | 内容 | 大小 | 触发条件 |
|------|------|------|----------|
| L1 | 最近 10 轮完整 | ~30K | 始终保留 |
| L2 | 当日摘要 | ~10K | 当日有对话 |
| L3 | 本周摘要 | ~5K | 本周有对话 |
| L4 | 长期记忆 | ~5K | 永久保留 |

### 动态调整规则

**当上下文 > 80K 时：**
1. 压缩 L2 → L3（当日摘要合并到本周摘要）
2. 如果仍 > 80K，压缩 L1（减少到最近 5 轮）
3. 如果仍 > 80K，压缩 L3 → L4（本周合并到长期）

**保证：**
- L1 永远完整保留（最近对话的连贯性）
- L4 永远存在（长期记忆不丢失）
- 总上下文永远 ≤ 100K

## 配置

### OpenClaw 配置
```json
{
  "agents": {
    "defaults": {
      "contextTokens": 100000,  // 100K 阈值
      "compaction": {
        "mode": "default"  // 自动压缩
      }
    }
  }
}
```

### 目录结构
```
~/.openclaw/
├── agents/main/sessions/     # 完整会话（自动）
│   └── <session-id>.jsonl    # 不删除，完整记录
│
└── workspace/memory/
    ├── MEMORY.md             # 长期记忆（L4）
    └── sessions/             # 压缩摘要
        ├── daily/            # 当日摘要（L2）
        │   └── qqbot-direct-YYYY-MM-DD.md
        ├── weekly/           # 本周摘要（L3）
        │   └── qqbot-week-YYYY-WW.md
        └── longterm/         # 长期摘要（L4备份）
            └── summary.md
```

## 工作流程

### 1. 会话进行中（自动）
- OpenClaw 自动跟踪 token 使用
- 超过 100K 时自动压缩历史
- 保留最近消息的完整性

### 2. 定时摘要（每 4 小时）
**触发条件：** 会话超过 10 轮

**执行内容：**
1. 读取当前会话的完整记录
2. 提取关键信息：
   - 用户意图/需求
   - 重要决策
   - 任务结果
   - 待办事项
3. 生成压缩摘要（< 2000 字）
4. 保存到 `memory/sessions/<channel>-<date>.md`
5. 更新 `MEMORY.md` 的关键信息

### 3. 历史检索（按需）
**当需要历史信息时：**
- `memory_search` 检索相关摘要
- `sessions_history` 查看完整记录
- 自动加载相关上下文
