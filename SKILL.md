# 通用合同审核技能

为企业提供可定制的商务合同系统化风险审查。

## 配置文件

审核规则存储在 `~/.openclaw/workspace/.contract-review-config.json`。

## 使用流程

### Step 0：首次使用 — 商业底线配置

**触发条件**：配置文件不存在时自动触发。

向用户依次提问以下 7 个模块，**每个问题都提供示例帮助用户理解**：

#### 模块 1：付款规则
```
请设定贵司的付款规则（可直接回复示例编号或自定义）：

1. 首款比例要求：____%  （示例：30%）
2. 首款支付期限：合同签署后 ____ 日内  （示例：10日）
3. 可接受的最长付款周期：____ 日  （示例：30日）
4. 验收逾期违约金：每天 ____‰ ，上限合同总价 ____%  （示例：0.5‰，3%）
```

#### 模块 2：责任上限
```
请设定贵司的责任边界：

1. 违约责任总额上限：不超过实际收款额的 ____%  （示例：100%，即实收费用总额）
2. 赔偿范围限定为：____  （示例：仅限直接经济损失）
3. 不承担的损失类型：____  （示例：间接损失、利润损失、惩罚性损害）
```

#### 模块 3：知识产权
```
请设定贵司的知识产权策略：

1. 标准产品是否提供源代码？  （示例：否）
2. 定制功能源代码如何处理？  （示例：可提供给客户，IP 归客户）
3. 是否允许"共同拥有知识产权"条款？  （示例：仅限定制/共创功能部分）
4. 产品名称要求：____  （示例：须与软著名称一致）
```

#### 模块 4：产品服务规则
```
请设定贵司的产品服务规则：

1. 订阅产品未续费是否自动终止？  （示例：是，license 停用）
2. 买断产品免费维保期：____  （示例：1年）
3. 维保费率：按产品金额的 ____%  （示例：15%）
4. 买断/定制产品是否提供免费升级？  （示例：否）
5. 项目交付周期上限：____  （示例：6个月）
```

#### 模块 5：保密与数据安全
```
请设定贵司的保密与数据安全规则：

1. 保密期限上限：____  （示例：项目周期+3年）
2. 是否接受"永久保密"条款？  （示例：否）
3. 数据泄露通知时限：____ 小时内  （示例：24-72小时）
4. 数据安全连带责任是否有上限？  （示例：是，不超过实收费用）
```

#### 模块 6：合规条款
```
请设定贵司的合规要求：

1. 违反廉洁条款的违约金：合同总价的 ____%  （示例：5%-10%）
2. 产品税率：____%  （示例：13%）
3. 服务税率：____%  （示例：6%）
4. 是否需要廉洁协议附件？  （示例：是）
```

#### 模块 7：服务方权益
```
请设定贵司的服务方权益保障：

1. 逾期 ____ 日可暂停服务  （示例：30日）
2. 暂停后超过 ____ 日仍未付款可终止合同  （示例：60日）
3. 履约保证金比例范围：合同总价的 ____% - ____%  （示例：5%-10%）
4. 联系人变更须提前通知：____ 个工作日  （示例：10日）
```

### Step 1：预审确认

读取合同文本后，先确认以下 8 项前置信息：

1. 合同类型（产品销售/技术实施/定制开发/订阅协议/混合）
2. 合同金额及与报价单差异
3. 是否涉及第三方分包/渠道分佣
4. 是否存在履约保证金要求
5. 是否涉及定制开发功能
6. 是否涉及源代码交付要求
7. 是否涉及数据安全合规要求
8. 是否涉及第三方知识产权授权

如有任一涉及，提示用户需提前评估风险。

### Step 2：否决条件扫描（7条即时触发）

对合同文本逐一检查以下否决条件，**命中任一条即输出【否决】**：

| # | 检查项 | 检查方式 |
|---|--------|----------|
| 1 | 侵犯己方知识产权（共有IP条款，超出配置允许范围） | 搜索"共同拥有""共同享有知识产权"，对照IP配置 |
| 2 | 无违约责任上限 | 搜索"赔偿不封顶""无上限赔偿" |
| 3 | 付款周期超过配置的最大天数 | 提取付款条款中的日期/天数，对照 `max_payment_cycle_days` |
| 4 | 己方须提供所有源代码（超出配置范围） | 搜索"源代码""源码交付""全部源码"，对照IP配置 |
| 5 | 保密期限超过配置上限或永久 | 搜索"永久保密""无期限保密"，对照保密配置 |
| 6 | 无限连带赔偿责任（无上限限定） | 搜索"无限连带""连带赔偿" |
| 7 | 数据安全责任无限连带（无上限） | 搜索数据安全条款中的连带责任，对照配置 |

### Step 3：七大类红线逐项审查

按以下类别逐条对照审查，详细规则见 [references/redlines.md](references/redlines.md)，所有数值均从配置文件读取：

1. **知识产权**：产品名称/源代码/IP归属/共有IP → 对照 `ip` 配置
2. **违约责任**：责任上限/直接损失原则 → 对照 `liability` 配置
3. **验收&回款**：退款规则/需求变更/首款/验收/付款周期/背靠背 → 对照 `payment` 配置
4. **产品&服务**：订阅终止/买断维保/升级/扩容/交付周期 → 对照 `product_service` 配置
5. **保密&数据安全**：保密期限/数据安全 → 对照 `confidentiality` 配置
6. **合规&廉洁**：反贿赂/税务发票 → 对照 `compliance` 配置
7. **服务方权益**：暂停服务/履约金/通知送达 → 对照 `service_rights` 配置

### Step 4：必备条款完整性检查

确认合同是否包含以下必备条款（缺失则为【警告】），所有数值来自配置：

- 违约责任上限（≤ 配置的责任上限比例）
- 赔偿范围限定为配置范围（排除配置的排除项）
- 首款 ≥ 配置的最低比例%（签署后配置的期限内）
- 付款周期 ≤ 配置的最长天数
- 验收环节 + 支付节点 + 验收标准附件
- 验收逾期违约金（每天配置的‰率，上限配置的%）
- 暂停服务权（逾期配置的天数）
- 送达条款（含电子送达）

如果是订阅合同，额外检查：
- 订阅自动终止条款（按配置）
- 扩容费用条款

如果是买断合同，额外检查：
- 维保期限制（按配置）
- 无"免费升级"承诺（按配置）

### Step 5：输出审核报告

```
📋 合同审核报告

基本信息：
- 合同类型：xxx
- 合同金额：xxx
- 审核依据：企业自定义商业底线规则

🚫 否决项（N/N）：
[如有，列出命中条款+原文+修改建议]

⚠️ 警告项（N/N）：
[如有，列出风险点+对应条款+修改建议]

✅ 必备条款检查：
[逐项列出通过/缺失，显示配置值作为依据]

📝 改进建议：
[其他非红线但建议优化的条款]

审核结论：✅ 通过 / ⚠️ 有条件通过 / 🚫 否决
```

## 关键词速查

审核时可使用 [references/keywords.md](references/keywords.md) 快速定位风险条款中的关键词。

## 配置管理

| 操作 | 说明 |
|------|------|
| 首次使用 | 自动触发引导问答 |
| 修改配置 | 用户说"修改合同审核配置"即可重新设定 |
| 查看配置 | 用户说"查看合同审核配置"显示当前规则 |
| 重置配置 | 用户说"重置合同审核配置"重新走引导流程 |
| 导出配置 | 配置文件可复制给团队统一使用 |
