## 完成的工作

### doc-to-ppt 技能升级 V2.0

**改造内容**：全流程 AI 驱动

**新流程**：
```
PDF/Word/PPT
     ↓
[AI 提取关键信息]  ← ai_extract_key_info.py
     ↓
[AI 规划幻灯片结构] ← 集成在 workflow_v2.py
     ↓
[AI 生成 HTML]     ← ai_generate_html.py
     ↓
[PNG → PPTX]       ← html2png_batch.py + png2pptx.py
```

**新增脚本**：
- `scripts/workflow_v2.py` - 主流程控制器
- `scripts/ai_extract_key_info.py` - Step 1: AI 深度理解文档

**核心改进**：
- 不是简单文本提取，AI 深度理解文档内容
- AI 提取：文档类型、核心观点、数据点、章节结构、叙事建议
- AI 规划：自动选择模板类型、决定页数、设计叙事逻辑
- AI 生成：为每页生成高质量 HTML（布局、图表、配色）

**测试结果**：
- 输入：已有的 slides_structure.json
- 输出：10 页 PPTX（1MB）
- 耗时：约 5 分钟（AI 生成 HTML 较慢，每页约 30-90 秒）
- 问题：slide 8 AI 超时，使用了 fallback 模板

**使用方式**：
```bash
python3 scripts/workflow_v2.py --file document.pdf --output slides.pptx
```

**待优化**：
- [ ] AI 生成 HTML 的超时处理（可增加重试）
- [ ] 并行生成 HTML（提速）
- [ ] 添加进度条

## 技术细节

### 超时问题
- `ai_generate_html.py` 的 `call_ai_agent()` 超时设置为 90 秒
- 超时后使用 `generate_fallback_html()` 生成基础模板
- 建议：增加重试机制或提高超时阈值

