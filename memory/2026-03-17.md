## doc2slides 进度环修复 (20:38 UTC+8)

**问题**：LLM 生成的进度环 `stroke-dashoffset="251.2"` 表示 0% 进度，但文字显示 100%

**根因**：prompt 不够明确，LLM 理解错误

**修复**：
- 更新 `llm_generate_html.py` 的 DASHBOARD 布局提示
- 明确公式：`stroke-dashoffset = 251.2 × (1 - 进度/100)`
- 100% 进度 → stroke-dashoffset="0"
- 90% 进度 → stroke-dashoffset="25.12"

**同时修复**：PYRAMID 布局缺少 flex 容器，导致金字塔和卡片没有左右对齐

## doc2slides 图文结合强制优化 (22:01 UTC+8)

**问题**：用户反馈"没有图文结合的布局"

**根因**：
1. 当 `data_points` 为空时，LLM 默认生成纯文字卡片
2. prompt 没有强制要求"每个页面必须有图形元素"

**解决方案**：
1. 创建 `strict_image_text_prompt.py` - 强制图文结合规则
2. 更新 `llm_generate_html.py` - 优先使用新 prompt
3. 新增规则：
   - 每个卡片必须有 SVG 图标（提供 10 种图标模板）
   - 每个页面必须有背景装饰（网格/渐变圆形/斜线）
   - 数据可视化 + 图标装饰结合

4. 修复进度环计算问题：
   - 预计算 `_stroke_dashoffset` 值
   - 直接传入 prompt，不再让 LLM 计算
   - 示例：33% → stroke-dashoffset="168.3"

**测试结果**：
- slide 1 (目录页)：背景网格 + 4 个卡片带图标 ✅
- slide 2 (内容页)：渐变圆形装饰 + 4 个 KPI 卡片带图标和进度环 ✅

**文件位置**：`~/.openclaw/workspace/skills/doc2slides/scripts/strict_image_text_prompt.py`
