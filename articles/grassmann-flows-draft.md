> 论文：*Attention Is Not What You Need: Grassmann Flows as an Attention-Free Alternative for Sequence Modeling*
> 作者：Chong Zhang

说实话，第一次看到这篇论文标题的时候，我差点以为又是哪个搞噱头的标题党——毕竟"Attention Is All You Need"可是深度学习领域最具辨识度的论文标题之一，敢直接对着干，胆子不小。

## 一、自注意力的"皇帝新衣"？

### 1.1 先回顾一下：自注意力到底在干什么

在 Transformer 诞生之前，处理序列数据（文本、语音、时间序列）主要靠 RNN 和 LSTM。它们有一个共同的痛点：处理第 t 个 token 的时候，信息是"一步一步传递"过来的——你看到第 t 个词，只能通过前一个隐藏状态"间接"地"看到"更早的词。这就像传话游戏，传到最后信息早就面目全非了。

2017 年，Vaswani 等人提出了 Transformer，核心就是**自注意力**（Self-Attention）。它的想法极其简单粗暴：干嘛一步步传？直接让每个 token 和所有其他 token"对话"不就行了？

用数学来说，对于输入序列 $X \in \mathbb{R}^{L \times d}$（L 是序列长度，d 是隐藏维度），标准的自注意力计算过程是这样的：

$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

这个 $QK^T$ 就是那个 L×L 的注意力矩阵——每个元素 $A_{ij}$ 表示第 i 个 token 对第 j 个 token 的"关注度"。然后拿这个权重矩阵去加权 $V$，得到新的表示。

**大白话版：** 想象一个会议桌，L 个人围坐一圈。自注意力就是让每个人同时跟所有人进行眼神交流——你边听边看，根据每个人的发言内容决定"谁的话对我最重要"，然后综合所有人的信息形成自己的新理解。这就是那个 L×L 矩阵的含义。

### 1.2 看起来完美，但代价是什么？

自注意力确实是好东西，但它有一个让人头疼的问题：**计算复杂度**。

那个 $QK^T$ 矩阵是 L×L 的。计算它需要 $O(L^2 d_k)$，再乘以 $V$ 又需要 $O(L^2 d_v)$。当序列长度 L 很大的时候（比如长文档、代码、DNA 序列），这个 L² 就会把你搞崩溃。

这就是为什么过去几年出现了那么多"线性注意力"变体：Linformer、Performer、Linear Transformer……它们的核心思路都是想办法把 $QK^T$ 矩阵"压缩"或者"近似"掉，避免显式计算那个巨大的 L×L 矩阵。

**但这些方案多少都有点"打补丁"的味道。** 它们保留了自注意力的基本框架，只是在计算上做了近似。而 Chong Zhang 的论文提出的是一个更根本的问题：我们真的需要这个框架吗？

### 1.3 张量提升：一种新的解读视角

这是整篇论文最精彩的部分。作者不是简单地说"自注意力不好，我换个方案"，而是从一个更高的数学视角重新解读了自注意力的本质。

作者指出，标准多头注意力可以理解为一种**张量提升**（tensor lifting）操作。什么意思呢？

假设你有 L 个隐藏向量 $h_1, h_2, \ldots, h_L \in \mathbb{R}^d$。自注意力做的事情是：对于每一对 $(h_i, h_j)$，通过 $Q, K, V$ 的线性变换，计算出一个标量注意力权重，然后用这个权重来混合信息。

但如果你换一个视角来看——如果你不做那个 softmax 归一化，直接看 $QK^T$ 这个矩阵本身，它本质上是一个从 $\mathbb{R}^{L \times d}$ 到 $\mathbb{R}^{L \times L}$ 的映射。你把一个"向量序列"提升成了一个"向量对的矩阵"。

**打个比方：** 想象你有一个二维照片（L×d 的矩阵），自注意力就像是用某种魔法把它"提升"成了一个三维的体数据（L×L 的矩阵加上各种变换）——信息量暴增了，表达能力也暴增了，但问题是，你很难再"看懂"这个三维结构里到底发生了什么。

用论文的术语来说：这种张量提升把隐藏向量映射到了成对交互的高维空间（pairwise interaction space），然后通过梯度下降来约束这个高维张量。机制表达能力极强，但**数学上不透明**（mathematically opaque）——因为经过多层叠加后，你几乎无法用一组有限的显式不变量来描述模型的行为。

**我的理解是：** 自注意力就像给了模型一台万能机器，它几乎可以做任何事（万能近似定理保证了这一点），但你完全不知道机器内部是怎么运作的。这不就是深度学习被诟病的"黑箱"问题的根源之一吗？

## 二、Grassmann 流形：一种"有结构"的替代方案

### 2.1 为什么需要流形？

如果自注意力的问题是"太自由、太无结构"，那自然的解决方案就是：**找一个有明确结构的数学对象来做替代。**

这就是流形（manifold）出场的时候了。

流形是什么？简单说，流形就是"局部像欧几里得空间，但全局有复杂拓扑结构"的数学对象。地球就是一个经典的例子：你在局部看，地面是平的（二维欧几里得空间），但全局来看，它是一个球面（有复杂拓扑的二维流形）。

论文选择的不是普通的流形，而是 **Grassmann 流形**（Grassmann manifold）。这是一种在代数几何和微分几何中非常重要的流形，也是线性代数的一种"升级版"。

### 2.2 Grassmann 流形的直观理解

$Gr(k, r)$ 定义为 $\mathbb{R}^r$ 中所有 k 维线性子空间的集合。

别被这个定义吓到。让我用一个最简单的例子来解释。

**一维的情况：** $Gr(1, 2)$ 就是 $\mathbb{R}^2$ 中所有一条直线穿过的原点的"方向"。每条过原点的直线由一个方向角 $\theta$ 确定，所以 $Gr(1, 2)$ 等价于一个圆——角度 $\theta$ 和 $\theta + \pi$ 表示同一条直线。你可以想象：站在原点，往四面八方看，每条视线就是一个一维子空间，所有视线的集合就是 $Gr(1, 2)$。

**二维的情况：** $Gr(2, 3)$ 就是 $\mathbb{R}^3$ 中所有过原点的平面。每个平面可以由两个基向量张成，所有平面的集合构成一个 Grassmann 流形。

**论文用的：** $Gr(2, r)$，也就是 $\mathbb{R}^r$ 中所有二维子空间的集合。每个二维子空间由两个向量张成，可以想象成一个嵌入在 r 维空间中的"小平面"。

### 2.3 为什么选 Grassmann 流形？

论文的论证逻辑是这样的：

1. 序列建模中，token 之间的交互本质上是一种**成对关系**——token i 和 token j 之间的关系
2. 自注意力用 L×L 矩阵来表示这种成对关系，但这个空间太大了（维度是 $L^2$），不可解释
3. 如果我们用**二维子空间**来表示"一对 token 之间的关系"，那么每对关系就对应 Grassmann 流形上的一个点
4. Grassmann 流形的维度是固定的（取决于 r），不随序列长度增长
5. 在这个流形上，我们有丰富的数学工具来分析和操作

**生活类比：** 想象你在组织一场相亲活动。传统自注意力的做法是：让每个人跟所有人一对一见面，记录每对之间的好感度（L×L 的矩阵）。而 Grassmann 流形的做法是：每对配对的人不是简单记一个"好感分数"，而是记录他们的"关系特征"——比如共同的兴趣爱好、价值观匹配度等。这些特征可以被编码成一个"关系向量"，所有可能的关系向量构成了一个有限维的流形。

### 2.4 Grassmann 流形上的一些关键性质

在继续讲架构之前，先补充几个重要的数学概念：

**维度：** $Gr(k, r)$ 的维度是 $k(r - k)$。对于论文用的 $k = 2$，维度是 $2(r - 2) = 2r - 4$。当 $r = 32$ 时，$Gr(2, 32)$ 的维度是 60。

**旋转不变性：** 一个二维子空间可以用很多组不同的基向量来表示，但子空间本身是不变的。就像一个平面可以用无数种坐标系来描述，但平面还是那个平面。这种不变性对于理解模型的"语义不变性"非常重要。

**几何结构：** Grassmann 流形上有自然的黎曼度量，可以计算"两个子空间之间的距离"。这使得我们可以在一个有明确几何意义的框架下比较不同 token 对之间的关系。

## 三、Plücker 坐标：把子空间变成向量

### 3.1 核心问题：如何"编码"一个子空间？

有了 Grassmann 流形的概念，下一个关键问题是：**怎么在计算机中表示一个二维子空间？**

一个直觉上最直接的方法是：用两个基向量。但问题是，基向量的选择有无穷多种——$(u, v)$ 和 $(u + v, v)$ 张成的是同一个子空间。我们需要一种"规范化的表示"，能唯一确定一个子空间。

这就是 **Plücker 坐标**（Plücker coordinates）出场的时候。

### 3.2 Plücker 坐标的数学定义

给定两个向量 $u, v \in \mathbb{R}^r$，它们张成的二维子空间的 Plücker 坐标是一个包含 $\binom{r}{2}$ 个分量的向量，每个分量是 $u$ 和 $v$ 对应分量的"行列式"：

$$p_{ij} = u_i v_j - u_j v_i, \quad \text{对所有 } 1 \leq i < j \leq r$$

这个向量 $p = (p_{12}, p_{13}, \ldots, p_{r-1,r}) \in \mathbb{R}^{\binom{r}{2}}$ 就是这个二维子空间的 Plücker 坐标。

**大白话解释：** 你可以把 Plücker 坐标理解为一个"子空间的指纹"。就像每个人都有独一无二的指纹一样，每个二维子空间也有独一无二的 Plücker 坐标（在归一化之后）。

### 3.3 Plücker 坐标的关键性质

**1. 不依赖于基的选择。** 如果你用 $(u', v')$ 张成同一个子空间（即 $u' = \alpha u + \beta v$，$v' = \gamma u + \delta v$，且 $\alpha\delta - \beta\gamma \neq 0$），那么新的 Plücker 坐标 $p' = (\alpha\delta - \beta\gamma) p$。也就是说，新的 Plücker 坐标只是原坐标的缩放——方向没变！归一化之后就完全一样了。

这就是为什么 Plücker 坐标能"抓住"子空间本身，而不是抓住某个特定的基。

**2. 维度有限且固定。** 当 $r = 32$ 时，$\binom{32}{2} = 496$。所以每个二维子空间用一个 496 维的向量就能表示。这个维度不随序列长度变化！

**3. Plücker 关系式。** 并非所有 $\mathbb{R}^{\binom{r}{2}}$ 中的向量都是合法的 Plücker 坐标。合法的坐标必须满足一组二次方程（称为 Plücker 关系）。这些关系式定义了 Plücker 嵌入像，是 Grassmann 流形在高维空间中的表示。

### 3.4 一个具体的例子

假设 $r = 3$，那么 $\binom{3}{2} = 3$。给定两个向量：

$$u = (u_1, u_2, u_3), \quad v = (v_1, v_2, v_3)$$

Plücker 坐标为：

$$p_1 = p_{12} = u_1 v_2 - u_2 v_1$$
$$p_2 = p_{13} = u_1 v_3 - u_3 v_1$$
$$p_3 = p_{23} = u_2 v_3 - u_3 v_2$$

注意到这个向量 $(p_1, p_2, p_3)$ 有一个重要性质：它恰好是 $u$ 和 $v$ 的**叉积**（cross product）！在三维空间中，两个向量的叉积的方向垂直于它们张成的平面，长度等于平行四边形的面积。所以 Plücker 坐标编码了子空间的"方向"和"大小"。

对于更高维的情况，Plücker 坐标就是叉积在高维的推广（外积/wedge product的分量）。

### 3.5 我的思考：为什么 Plücker 坐标特别适合 token 对？

我觉得这个选择很巧妙。Plücker 坐标本质上是两个向量的**外积**的分量——它抓住了两个向量的"相对位置"和"张成的面积"，而不是每个向量的绝对信息。

在语义层面，这不正是我们想要的吗？当我们分析两个 token 之间的关系时，我们关心的是它们之间的"相对语义"——比如"国王"和"王后"之间的关系，而不是"国王"这个概念本身包含了什么信息。

具体来说：
- 如果两个 token 的表示几乎平行（语义相似），那么 Plücker 坐标的模长很小
- 如果两个 token 的表示正交（语义无关），那么 Plücker 坐标的模长较大
- 如果两个 token 的表示反平行（语义对立），Plücker 坐标的模长也较大

这种"只看相对关系，不看绝对位置"的特性，正是自注意力在做的事情（$Q_i \cdot K_j$ 也是在衡量相对相似度），但 Plücker 坐标提供了更丰富、更有结构的表示。

## 四、因果 Grassmann 层：架构详解

好了，数学铺垫够了，让我们看看实际的模型架构。论文提出的核心模块叫 **因果 Grassmann 层**（Causal Grassmann Layer），它是 Transformer 中多头注意力层的"替代品"。

### 4.1 整体架构一览

因果 Grassmann 层的处理流程如下：

```
输入 h_t ∈ ℝ^d
    ↓
(1) 线性降维: z_t = W_red · h_t ∈ ℝ^r
    ↓
(2) 多尺度局部配对: 对每个偏移 Δ ∈ W, 形成 (z_t, z_{t+Δ})
    ↓
(3) Plücker 编码: 计算每对的 Plücker 坐标, 归一化
    ↓
(4) 投影回模型空间: g_t = W_plü · [各偏移的 Plücker 特征]
    ↓
(5) 门控融合: h_t' = gate · g_t + (1 - gate) · h_t
    ↓
(6) FFN + 残差 + LayerNorm (标准后处理)
```

一步一步来看。

### 4.2 第一步：线性降维

$$z_t = W_{\text{red}} h_t + b_{\text{red}}, \quad W_{\text{red}} \in \mathbb{R}^{r \times d}$$

这里把隐藏状态从 $\mathbb{R}^d$ 投影到 $\mathbb{R}^r$，其中 $r \ll d$。论文中 $d = 256$，$r = 32$。

为什么要先降维？

两个原因：
1. **控制 Plücker 坐标的维度：** Plücker 坐标的维度是 $\binom{r}{2}$，如果 $r = d = 256$，那就是 $\binom{256}{2} = 32640$ 维，太大了。降到 $r = 32$ 后，$\binom{32}{2} = 496$ 维，可控得多。
2. **信息压缩/去噪：** 降维矩阵 $W_{\text{red}}$ 在训练中学习，它会自动学习哪些信息对下游的 Grassmann 交互最重要。这是一种"瓶颈"设计，强迫模型只保留最关键的信息。

**类比：** 就像你去看一场交响乐，256 件乐器同时在演奏。但你不需要把每一件乐器的每一个音符都记下来——你只需要记下 32 个"关键乐句"（由降维矩阵选择），然后用这 32 个乐句来理解整场演出的结构和情感。

### 4.3 第二步：多尺度局部配对

这一步是因果 Grassmann 层的关键创新之一。

给定降维后的表示 $z_t \in \mathbb{R}^r$，对于每个位置 $t$ 和每个偏移量 $\Delta \in \mathcal{W}$，形成一对 $(z_t, z_{t+\Delta})$。

论文使用的偏移窗口集合是：

$$\mathcal{W} = \{1, 2, 4, 8, 12, 16\}$$

也就是说，每个 token 会跟它后面 1 个、2 个、4 个、8 个、12 个和 16 个位置的 token 配对。

**为什么要多尺度？** 这是为了捕获不同距离的依赖关系：
- $\Delta = 1, 2$：捕获局部/相邻关系（类似 bigram、trigram 信息）
- $\Delta = 4, 8$：捕获中等距离的上下文（跨越短语/从句）
- $\Delta = 12, 16$：捕获较远距离的信息

这让我想到了计算机视觉中的多尺度特征提取——ResNet 等网络在不同分辨率上提取特征，然后融合。本质上是一样的道理：不同尺度的信息有不同的价值，全部用上才能获得全面的理解。

**因果性：** 注意，这里的配对是"向后看"的——$z_{t+\Delta}$ 只出现在 $z_t$ 的后面（时间上更晚）。这保证了因果性（causality），即当前 token 的表示只依赖于过去的信息，不会"偷看"未来的 token。这在语言建模中至关重要——你不能用后面的词来预测前面的词！

### 4.4 第三步：Plücker 编码

对每一对 $(z_t, z_{t+\Delta})$，计算 Plücker 坐标：

$$p_t^{(\Delta)} = z_t \wedge z_{t+\Delta}$$

用分量来写就是：

$$p_{t,ij}^{(\Delta)} = z_{t,i} \cdot z_{t+\Delta,j} - z_{t,j} \cdot z_{t+\Delta,i}, \quad \forall 1 \leq i < j \leq r$$

然后进行归一化：

$$\hat{p}_t^{(\Delta)} = \frac{p_t^{(\Delta)}}{\|p_t^{(\Delta)}\|_2}$$

归一化确保了 Plücker 坐标只编码"子空间的方向"（两个 token 之间的相对关系），而不编码"子空间的大小"（两个 token 表示的绝对幅度）。

**代码视角：** 在实际实现中，这一步可以通过高效的向量运算来完成。对于每个偏移 $\Delta$，计算 $z_t$ 和 $z_{t+\Delta}$ 的所有两两分量差积，得到一个 $L \times \binom{r}{2}$ 的张量。

**计算复杂度分析：** 对于每个位置 $t$ 和每个偏移 $\Delta$，计算 $\binom{r}{2}$ 个分量。共有 $L \times |\mathcal{W}|$ 个 (位置, 偏移) 对。所以总计算量是 $O(L \cdot |\mathcal{W}| \cdot \binom{r}{2}) = O(L \cdot |\mathcal{W}| \cdot r^2)$。当 $r$ 和 $|\mathcal{W}|$ 固定时，这是关于 L 线性的！

### 4.5 第四步：投影回模型空间

现在我们有了一系列归一化的 Plücker 特征 $\hat{p}_t^{(\Delta)}$，每个是 $\binom{r}{2}$ 维的。下一步是把它们映射回模型的隐藏维度 $d$。

具体做法是：先把所有偏移的 Plücker 特征拼接起来，然后用一个线性层投影：

$$\tilde{g}_t = W_{\text{plü}} \cdot \text{concat}\left(\hat{p}_t^{(\Delta_1)}, \hat{p}_t^{(\Delta_2)}, \ldots, \hat{p}_t^{(\Delta_m)}\right)$$

其中 $W_{\text{plü}} \in \mathbb{R}^{d \times m \cdot \binom{r}{2}}$，$m = |\mathcal{W}|$ 是偏移的数量。

对于 $r = 32$ 和 $m = 6$，拼接后的维度是 $6 \times 496 = 2976$。然后 $W_{\text{plü}}$ 把这个 2976 维向量映射到 $d = 256$ 维。

**这一步的作用：** Plücker 坐标在 Grassmann 流形上编码了"子空间"信息，但这些信息需要被"翻译"回模型的"语言"（$\mathbb{R}^d$ 空间），才能跟后续的网络层交互。这个线性层就充当了"翻译官"的角色。

### 4.6 第五步：门控融合

最后一步是把几何特征融合回原始的隐藏状态。论文使用了一个门控机制（gating mechanism），这在 LSTM/GRU 中也很常见：

$$\text{gate}_t = \sigma\left(W_{\text{gate}} \cdot [h_t; g_t] + b_{\text{gate}}\right)$$

$$h_t' = \text{gate}_t \odot g_t + (1 - \text{gate}_t) \odot h_t$$

其中 $\sigma$ 是 sigmoid 函数，$[h_t; g_t]$ 是拼接，$\odot$ 是逐元素乘法。

门控机制的直觉是：对于每个维度，模型独立地决定"保留多少原始信息"和"注入多少新的几何信息"。如果 $\text{gate}_t$ 接近 1，就主要使用几何特征；如果接近 0，就主要保留原始状态。

**为什么要门控？** 不是所有的 token 都需要"更新"自己的理解。有些词的含义很明确（比如"苹果"在大多数语境下就是那个水果），而有些词的含义需要根据上下文来确定（比如"bank"到底是银行还是河岸）。门控机制让模型能够灵活地选择何时"听取"Grassmann 交互的信息，何时保持自己的判断。

### 4.7 第六步：标准后处理

门控融合之后，跟标准的 Transformer 一样，后面接一个前馈网络（FFN），加上残差连接和层归一化：

$$\text{output}_t = \text{LayerNorm}\left(h_t' + \text{FFN}(h_t')\right)$$

FFN 通常是一个两层 MLP：

$$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$

这一部分没有什么特别的，保持跟 Transformer 一样就好。

## 五、完整模型架构：GrassmannLM

### 5.1 把因果 Grassmann 层堆起来

论文中的语言模型 GrassmannLM 的整体结构跟 Transformer 很类似：

1. **Token 嵌入层**：标准的词嵌入 + 位置编码
2. **N 个因果 Grassmann 层**：每个层包含上述的完整流程（降维 → 配对 → Plücker 编码 → 投影 → 门控融合 → FFN）
3. **输出层**：线性层 + softmax，预测下一个 token

论文测试了两种配置：
- **6 层**（约 1300 万参数）
- **12 层**（约 1800 万参数）

### 5.2 跟 Transformer 的完整对比

为了更清晰地理解两种架构的差异，我用一个表来对比：

| 特性 | Transformer | GrassmannLM |
|------|------------|-------------|
| 核心交互机制 | 自注意力（$QK^T$ 矩阵） | Grassmann 流形上的子空间交互 |
| 序列间信息传递 | 通过 L×L 注意力矩阵 | 通过多尺度局部配对 + Plücker 编码 |
| 空间结构 | 无结构（平坦的 $\mathbb{R}^{L \times L}$ 空间） | 有结构（Grassmann 流形，有几何意义） |
| 计算复杂度 | $O(L^2d + Ld^2)$ | $O(Ld^2)$（关于 L 线性） |
| 内存占用 | $O(L^2)$（需要存储注意力矩阵） | $O(L)$ |
| 可解释性 | 低（注意力权重能提供一些线索，但整体不透明） | 高（Plücker 坐标有明确的几何意义） |
| 数学基础 | 线性代数 + softmax | 代数几何 + 微分几何 |
| 全局长程依赖 | 天然支持 | 通过堆叠多层逐步传播 |

### 5.3 复杂度分析的深入

让我们更仔细地分析一下 GrassmannLM 的计算复杂度。

因果 Grassmann 层的计算量主要来自以下部分：

1. **线性降维：** $W_{\text{red}} \in \mathbb{R}^{r \times d}$，对每个 token 做一次矩阵乘，$O(Lrd)$

2. **Plücker 编码：** 对每对 (位置, 偏移) 计算 $\binom{r}{2}$ 个分量，$O(L \cdot m \cdot r^2 / 2)$

3. **投影回模型空间：** $W_{\text{plü}} \in \mathbb{R}^{d \times m \cdot \binom{r}{2}}$，$O(L \cdot d \cdot m \cdot r^2 / 2)$

4. **门控融合：** $W_{\text{gate}} \in \mathbb{R}^{d \times 2d}$，$O(L \cdot 2d^2)$

5. **FFN：** 标准的两层 MLP，$O(L \cdot d \cdot d_{\text{ff}})$

注意到了吗？**没有任何一项包含 $L^2$！** 所有计算都是关于 L 线性的。这是因为我们从未构造一个 L×L 的矩阵——每个 token 只跟固定数量的"邻居"（通过固定偏移集合）交互。

**但是**，论文也诚实地指出，当前的 PyTorch 实现中，Plücker 坐标的计算是用循环实现的（逐对计算行列式），效率不高。而 GPU 上的自注意力（如 FlashAttention）经过了大量优化，实际上运行速度很快。所以虽然在理论上 GrassmannLM 的复杂度是线性的，但在当前的实现下，它的实际运行速度可能反而比优化后的 Transformer 慢。

这是一个非常诚实的评估。论文没有夸大性能优势，而是清楚地指出了理论和实践之间的差距。

## 六、实验结果：现实检验

### 6.1 Wikitext-2 语言建模

语言建模是序列建模的"标准考题"。模型需要根据前面的文本来预测下一个词，困惑度（perplexity）越低越好。

| 模型 | 层数 | 参数量 | 验证困惑度 |
|------|------|--------|-----------|
| TransformerLM | 6 | ~13M | 241.0 - 253.6 |
| **GrassmannLM** | 6 | ~13M | **275.7** |
| TransformerLM | 12 | ~18M | 235.2 |
| **GrassmannLM** | 12 | ~18M | **261.1** |

**解读：**

坦白说，数字不太好看。在相同参数量下，GrassmannLM 的困惑度比 Transformer 高大约 10-15%。

但是，我觉得有几点值得注意：

1. **这是一个全新的架构。** Transformer 经过多年的研究和优化，各种技巧（预归一化、GELU、更好的初始化等）都已经很成熟了。而 GrassmannLM 是第一次尝试，还有大量的超参数调优空间。

2. **差距在缩小。** 从 6 层到 12 层，Transformer 的困惑度从 ~247 降到 235（提升约 5%），而 GrassmannLM 从 276 降到 261（提升约 5.4%）。深度增加对 GrassmannLM 的帮助更大，这说明架构本身有深度可扩展的潜力。

3. **完全没有注意力！** 在完全不使用自注意力的情况下达到这个水平，本身就是一个有意义的结果。它证明了自注意力不是强性能的必要条件。

### 6.2 SNLI 自然语言推理

自然语言推理（NLI）是另一个重要任务：给定一个前提（premise）和一个假设（hypothesis），判断它们之间的关系是蕴含（entailment）、矛盾（contradiction）还是中立（neutral）。

论文的实验设置很巧妙：**不替换整个模型**，而是只替换"头部"。具体来说，使用预训练好的 DistilBERT 作为骨干网络（feature extractor），然后分别用 Transformer 头和 Grassmann-Plücker 头来做分类。

| 头部 | 验证准确率 | 测试准确率 |
|------|-----------|-----------|
| Transformer 头 | 0.8545 | 0.8511 |
| **Grassmann-Plücker 头** | **0.8550** | **0.8538** |

**这个结果很有意思！** Grassmann 头反而**略优**于 Transformer 头。

为什么会这样？我猜测原因是：NLI 任务本质上就是在判断两个句子之间的"关系"——蕴含、矛盾还是中立。这恰好就是 Plücker 坐标擅长的东西：编码两个对象之间的几何关系。在 NLI 中，两个句子的表示可以看作 Grassmann 流形上的两个"点"，它们之间的距离和方向自然地编码了它们的关系。

相比之下，自注意力虽然通用性更强，但在这类"关系判断"任务上可能反而"杀鸡用牛刀"了。

### 6.3 我的解读

综合两个实验的结果，我的看法是：

**GrassmannLM 目前还不能取代 Transformer，但它展示了一个有价值的研究方向。**

- 在语言建模这种"需要全局信息"的任务上，由于只使用局部配对，信息需要通过多层逐步传播，在较浅的模型中可能无法覆盖足够长的距离，导致性能有差距。
- 在 NLI 这种"关系判断"任务上，Grassmann 几何表示反而展现出独特的优势。
- 线性复杂度是一个重要优势，尤其在处理长序列时。

这让我想到了 Mamba 和 State Space Models（SSM）的轨迹：最初的 SSM 性能也不如 Transformer，但随着架构改进（选择性 SSM、Mamba 等），逐渐缩小了差距，并在特定场景下展现了优势。GrassmannLM 可能也在类似的轨道上。

## 七、深度讨论：从"张量提升"到"流形演化"

### 7.1 自注意力的"不可解释性"到底意味着什么？

论文反复提到自注意力的"不可解释性"（inexplicability）。但这个概念需要更深入地理解。

在数学中，一个系统是"可解释的"通常意味着：你可以用一组有限的不变量来描述系统的状态和行为。比如，一个刚体的运动可以用位置、速度、角速度等有限个量来完全描述——这就是可解释的。

但自注意力中，信息在 L×L 的矩阵中流动，经过多层变换后，你几乎无法用任何有限的不变量族来"总结"模型学到了什么。每一层都在做一种无结构的张量变换，就像在一个没有地图的迷宫里随机行走——走一万步之后，你很难用简单的语言描述"你到了哪里"。

**这不仅仅是一个"理解模型"的问题。** 它也影响模型泛化、鲁棒性和可控性。一个不可解释的模型，你很难预测它在分布外数据上的行为。

### 7.2 Grassmann 流形如何解决这个问题？

Grassmann 流形提供了一个"有结构"的信息交互空间：

1. **维度有限：** $Gr(2, r)$ 的维度是 $2r - 4$，不随序列长度增长
2. **有几何意义：** 每个点代表一个二维子空间，有明确的数学含义
3. **有不变量：** 比如子空间的维数、行列式、Plücker 关系式等
4. **有度量：** 可以计算子空间之间的距离，进行有意义的比较

在这种框架下，信息传播可以被理解为"语义流形上的几何演化"——每处理一个 token，语义子空间就在流形上做一次"形变"（deformation），这些形变累积起来，就形成了模型对整个序列的理解。

**这是一种非常优雅的视角。** 它把"序列理解"从一个纯粹的统计/优化问题，提升到了一个几何问题。

### 7.3 通用的近似定理还成立吗？

一个自然的问题是：既然我们把信息交互限制在了 Grassmann 流形上（而不是无结构的张量空间），表达能力会下降吗？

答案是：**理论上不会。** 因为通用近似定理适用于任何足够"丰富"的空间。Grassmann 流形虽然是有限维的，但通过：
1. 多尺度配对（6 种偏移）
2. 多层堆叠（6-12 层）
3. 非线性门控 + FFN

模型有足够的表达能力来近似任意函数。

**关键区别在于：** 在 Transformer 中，近似发生在一个"没有明确结构"的高维空间中；而在 GrassmannLM 中，近似发生在一个"有明确几何结构"的有限维流形上。后者的好处是，我们可以更有信心地分析、理解和控制模型的近似过程。

打个比方：两者都能到达目的地，但一个是坐着没有导航的热气球飘过去的（可能飘得很快，但你不知道会经过哪些地方），另一个是沿着有地图标记的公路开过去的（路线有结构，你能知道经过了哪里）。

## 八、局部配对 vs 全局注意力：信息传播的"接力赛"

### 8.1 一个关键问题：局部配对怎么捕获长距离依赖？

自注意力最强大的能力之一是直接捕获任意距离的依赖——第 1 个词可以直接"看到"第 1000 个词。但 GrassmannLM 只用了局部配对（最大偏移 16），怎么捕获长距离依赖？

答案是：**通过多层堆叠，信息像接力赛一样逐层传播。**

假设模型有 12 层，每层的最大偏移是 16。那么：
- 第 1 层：每个 token 可以看到 16 步以内的信息
- 第 2 层：由于第 1 层已经把 16 步内的信息混合到了当前 token 中，第 2 层实际上可以间接"看到"32 步以内的信息
- 第 12 层：理论上可以间接覆盖 $12 \times 16 = 192$ 步的信息

当然，这只是理论上限。实际的信息传播效率取决于：
- 门控机制是否有效地传递了远距离信息
- 多个偏移尺度的组合是否覆盖了足够的距离
- 训练过程中是否学到了有效的信息传递策略

### 8.2 跟卷积和循环的对比

这种"局部配对 + 层间传播"的设计，让我想到了两个经典的方法：

**卷积网络（CNN）：** 也是通过局部感受野 + 层间堆叠来扩大覆盖范围。GrassmannLM 的多尺度配对有点类似"膨胀卷积"（dilated convolution），用不同的"步长"来快速扩大感受野。

**循环网络（RNN）：** 也是通过逐步传递信息来覆盖整个序列。但 RNN 是串行的（必须先处理完位置 t，才能处理位置 t+1），而 GrassmannLM 的因果层可以并行处理所有位置。

GrassmannLM 结合了两者的优点：
- 像 CNN 一样可以并行处理
- 像膨胀卷积一样用多尺度快速扩大覆盖范围
- 但不像 CNN 那样受限于固定的卷积核大小

### 8.3 为什么不用更大的偏移？

一个自然的问题是：为什么偏移集合 $\mathcal{W} = \{1, 2, 4, 8, 12, 16\}$ 的最大值只有 16？用更大的偏移（比如 32、64、128）不就能更快地覆盖长距离了吗？

可能的原因有几个：

1. **计算量：** 更多的偏移意味着更多的 Plücker 坐标计算。虽然仍然关于 L 线性，但常数因子会增大。

2. **过拟合风险：** 在小数据集上（如 Wikitext-2 只有约 200 万训练 token），过多的参数可能导致过拟合。

3. **信息稀释：** 距离太远的 token 之间的关系可能噪声较大，直接编码反而可能引入干扰。

但我觉得，更大的偏移集和更多的层数可能是提升 GrassmannLM 性能的一个方向。论文也提到了这一点作为未来的工作。

## 九、实现细节：如果你要自己复现

### 9.1 Plücker 坐标的计算实现

Plücker 坐标的核心计算是外积。对于 $u, v \in \mathbb{R}^r$，我们需要计算所有 $u_i v_j - u_j v_i$（$i < j$）。

一种高效的实现方式是：

```python
def plucker_coords(u, v):
    """
    u, v: (..., r)
    returns: (..., C(r,2))
    """
    # 利用外积的不对称部分
    outer = u[..., :, None] * v[..., None, :]  # (..., r, r)
    anti_sym = outer - outer.transpose(-1, -2)  # 取反对称部分
    # 取上三角部分（i < j）
    indices = torch.triu_indices(r, r, offset=1)
    return anti_sym[..., indices[0], indices[1]]
```

对于 $r = 32$，输出维度是 496。

**性能优化提示：** 上面的实现使用了矩阵乘法和索引操作，在 GPU 上可以批量化处理。但论文指出，当前的实现还有很大的优化空间——特别是可以利用 FlashAttention 类似的技术来减少内存访问。

### 9.2 完整的因果 Grassmann 层伪代码

```python
class CausalGrassmannLayer(nn.Module):
    def __init__(self, d, r, windows, d_ff):
        super().__init__()
        self.r = r
        self.windows = windows  # e.g., [1, 2, 4, 8, 12, 16]
        
        # (1) 线性降维
        self.W_red = nn.Linear(d, r)
        
        # (4) 投影回模型空间
        pluecker_dim = r * (r - 1) // 2
        self.W_plu = nn.Linear(len(windows) * pluecker_dim, d)
        
        # (5) 门控融合
        self.W_gate = nn.Linear(2 * d, d)
        
        # (6) FFN
        self.ffn = nn.Sequential(
            nn.Linear(d, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d)
        )
        self.norm1 = nn.LayerNorm(d)
        self.norm2 = nn.LayerNorm(d)
    
    def forward(self, h):
        # h: (L, d)
        L = h.size(0)
        
        # (1) 线性降维
        z = self.W_red(h)  # (L, r)
        
        # (2) + (3) 多尺度配对 + Plücker 编码
        pluecker_features = []
        for delta in self.windows:
            # 形成配对 (z_t, z_{t+delta})，注意因果性
            z_current = z[:-delta]   # (L-delta, r)
            z_future = z[delta:]     # (L-delta, r)
            
            # 计算 Plücker 坐标
            p = plucker_coords(z_current, z_future)  # (L-delta, C(r,2))
            p = F.normalize(p, dim=-1)  # 归一化
            
            # Pad 回 L 的长度（前面补零）
            p_padded = F.pad(p, (0, 0, delta, 0))  # (L, C(r,2))
            pluecker_features.append(p_padded)
        
        # 拼接所有偏移的特征
        all_features = torch.cat(pluecker_features, dim=-1)  # (L, m*C(r,2))
        
        # (4) 投影回模型空间
        g = self.W_plu(all_features)  # (L, d)
        
        # (5) 门控融合
        gate = torch.sigmoid(self.W_gate(torch.cat([h, g], dim=-1)))
        h_prime = gate * g + (1 - gate) * h
        
        # 残差 + LayerNorm
        h_prime = self.norm1(h + h_prime)
        
        # (6) FFN + 残差 + LayerNorm
        out = self.norm2(h_prime + self.ffn(h_prime))
        
        return out
```

### 9.3 关键超参数

论文中使用的超参数：

| 参数 | 值 | 说明 |
|------|---|------|
| d（隐藏维度） | 256 | 模型的主要隐藏维度 |
| r（降维后维度） | 32 | Plücker 编码的工作空间维度 |
| $\mathcal{W}$（偏移窗口） | {1,2,4,8,12,16} | 多尺度局部配对的偏移量 |
| d_ff（FFN 维度） | 1024 | 前馈网络的中间维度 |
| 层数 | 6 或 12 | 因果 Grassmann 层的数量 |
| 学习率 | 5e-4 | AdamW 优化器 |
| 权重衰减 | 0.01 | AdamW 的 L2 正则化 |

## 十、Grassmann 流形的进一步思考

### 10.1 为什么选 k=2？

论文使用 $Gr(2, r)$，即二维子空间。为什么不选更高维的子空间（$k = 3, 4, \ldots$）？

原因可能有几个：

1. **计算量：** $Gr(k, r)$ 的 Plücker 坐标维度是 $\binom{r}{k}$，这随 k 增长非常快。对于 $r = 32$：
   - $k = 2$：$\binom{32}{2} = 496$
   - $k = 3$：$\binom{32}{3} = 4960$
   - $k = 4$：$\binom{32}{4} = 35960$

   $k = 3$ 时维度已经是 $k = 2$ 的 10 倍，计算量也相应增加。

2. **二阶交互的充足性：** 很多研究表明，二阶交互（成对关系）已经能捕获大部分有用的信息。三阶和更高阶的交互虽然理论上更丰富，但边际收益递减。

3. **二阶的优雅性：** $k = 2$ 时，Plücker 坐标有一个特别优雅的形式（类似叉积），几何直觉也最清晰。

但我认为，探索 $k = 3$ 或混合不同 k 值的架构是一个值得研究的方向。

### 10.2 Grassmann 流形上的"流"（Flows）

论文标题中的"Grassmann Flows"值得特别讨论。什么是"流"？

在微分几何中，流（flow）是指一个连续的、随时间演化的变换。在 Grassmann 流形的语境下，信息处理过程可以被理解为：随着序列位置从左到右推进，语义子空间在流形上连续地"流动"和"变形"。

这跟 RNN 中的"隐藏状态演化"是类似的，但关键区别在于：
- RNN 的隐藏状态在 $\mathbb{R}^d$（平坦空间）中演化
- GrassmannLM 的子空间在 $Gr(2, r)$（弯曲流形）中演化

弯曲流形上的演化有更丰富的结构：曲率、测地线、平行输运等概念都可以被利用。这为未来的研究提供了丰富的数学工具。

### 10.3 跟其他无注意力架构的关系

近几年涌现了不少无注意力或减少注意力的架构：

1. **Mamba / SSM**：用状态空间模型替代自注意力，通过隐式状态来"记住"过去的信息
2. **RWKV**：结合了 RNN 和 Transformer 的线性注意力变体
3. **MLP-Mixer**：纯粹用多层感知机做序列混合
4. **Hyena / StripedHyena**：用长卷积替代自注意力

GrassmannLM 跟这些方法的不同之处在于它的**数学基础**。Mamba 基于线性系统理论，RWKV 基于线性注意力，MLP-Mixer 纯靠 MLP 的万能近似能力。而 GrassmannLM 基于的是**代数几何**——它的信息交互发生在有明确几何结构的流形上。

这意味着 GrassmannLM 天然拥有更丰富的数学分析工具。理论上，你可以用微分几何的工具来分析信息如何在流形上流动，用代数几何的工具来分析子空间的性质和变换。

### 10.4 潜在的研究方向

基于论文的讨论和我自己的思考，我觉得以下几个方向特别有前景：

**1. 更丰富的局部不变量。** 当前只用了 Plücker 坐标（本质上是外积），还可以加入行列式、内积、角度等几何不变量，丰富每对 token 的交互表示。

**2. 自适应偏移选择。** 当前的偏移集合是固定的。可以让模型根据输入动态选择偏移量（类似 Mamba 的选择性机制）。

**3. 全局不变量的构建。** 这是论文明确提到的未来方向。通过在局部 Plücker 特征上构建全局的池化或聚合操作，可以实现对整个序列的全局理解。

**4. 混合架构。** 把 Grassmann 层和自注意力层混合使用——用 Grassmann 层处理局部交互，用少量注意力层处理全局依赖。这种"8:2"的混合可能既保持线性复杂度，又不丢失长距离信息。

**5. 硬件优化。** 开发针对 Plücker 坐标计算的高效 GPU kernel，缩小理论复杂度和实际性能之间的差距。

## 十一、我的思考与评价

### 11.1 这篇论文的价值在哪里？

我觉得这篇论文最大的价值不在于"GrassmannLM 跑赢了 Transformer"——事实上它目前还没跑赢。它的真正价值在于：

**提供了一个全新的视角来理解序列建模。**

这个视角是：序列理解本质上是一种几何演化，而不是统计优化。Token 之间的关系可以被编码为几何对象（子空间），信息传播可以被理解为流形上的变形。

这种视角的好处是：
1. **可分析性：** 有成熟的数学工具可以用来理解和分析
2. **可解释性：** 每个计算步骤都有明确的几何含义
3. **可组合性：** 不同的几何结构可以灵活组合

### 11.2 为什么我觉得这个方向值得关注？

过去几年，深度学习的发展很大程度上是"工程驱动"的——更好的优化器、更大的模型、更多的数据、更快的硬件。理论理解一直在追赶实践。

Grassmann 流形的方法代表了一种"理论驱动"的尝试——不是先堆实验、再事后解释，而是从数学原理出发，设计一种有明确结构的架构。

这种尝试虽然短期可能不如工程优化来得有效，但长期来看，可能对"让深度学习变得可理解、可控、可信赖"更有帮助。

### 11.3 它的局限性

说了这么多好的，也得说说局限：

1. **性能差距是真实的。** 在 Wikitext-2 上 10-15% 的差距不小，说明当前架构的信息传播效率还不够高。

2. **理论优势还未充分转化为实践优势。** 线性复杂度在理论上很好，但实际运行速度反而更慢——这说明实现优化还有很大的空间。

3. **缺乏大规模实验。** 论文只在两个基准测试上做了实验，都是小规模任务。在更大规模的数据和模型上效果如何，还未知。

4. **多尺度偏移的选择缺乏理论指导。** $\{1, 2, 4, 8, 12, 16\}$ 是怎么选的？有没有更优的选择？论文没有给出充分的理论分析。

### 11.4 更大的图景

从更大的视角来看，我觉得这篇论文属于一个正在兴起的研究趋势：**用结构化的数学对象来替代无结构的神经网络组件。**

类似的趋势包括：
- Lie 群在旋转等变网络中的应用
- 图神经网络中的拓扑结构
- 流形正则化在表示学习中的应用
- 辛几何在 Hamiltonian 神经网络中的应用

这些工作的共同点是：**不是盲目增加模型容量，而是利用数学结构来提高效率和可解释性。**

GrassmannLM 就是这个趋势在序列建模领域的一个具体实例。

## 十二、动手指南：如果你想尝试

### 12.1 复现的步骤

如果你想自己复现论文的实验，建议按以下步骤进行：

1. **先实现 Plücker 坐标的计算函数**，确保理解了这个核心操作
2. **实现单个因果 Grassmann 层**，在随机数据上验证前向/反向传播
3. **用 Grassmann 层替换 Transformer 的注意力层**，在小数据集（如 PTB）上做快速验证
4. **在 Wikitext-2 上训练完整模型**，对比 Transformer baseline
5. **在 SNLI 上尝试 Grassmann 头部**，验证论文的 NLI 结果

### 12.2 关键注意事项

- **归一化很重要：** Plücker 坐标在计算后必须归一化，否则子空间的大小会干扰学习
- **因果 padding：** 在序列的起始位置，后面的偏移对会"超出"序列长度，需要正确处理
- **学习率敏感性：** Grassmann 层中的参数可能对学习率比 Transformer 更敏感，需要仔细调优
- **梯度流：** Plücker 坐标涉及到行列式计算，可能存在梯度消失/爆炸的风险

### 12.3 扩展实验建议

如果你想在这篇论文的基础上做进一步研究，我建议：

1. **尝试不同的偏移集合：** 比如 $\{1, 2, 4, 8, 16, 32\}$（纯指数增长）或 $\{1, 3, 5, 7, 9, 11\}$（等差数列）
2. **探索混合架构：** 在 Grassmann 层之间穿插少量注意力层
3. **更大的数据集：** 在 WikiText-103 或 C4 上验证
4. **消融实验：** 去掉某个偏移、改变降维维度 r、去掉门控机制等，看各组件的贡献

## 十三、总结对比

### 13.1 一图总结

为了方便对比，我用一个表格来总结两种架构的核心差异：

| 维度 | Transformer | GrassmannLM |
|------|------------|-------------|
| **核心思想** | 让每个 token 直接跟所有 token 对话 | 用流形上的子空间编码局部关系 |
| **数学基础** | 线性代数 + softmax | 代数几何（Grassmann 流形 + Plücker 坐标） |
| **信息交互空间** | $\mathbb{R}^{L \times L}$（无结构张量空间） | $Gr(2, r)$（有限维弯曲流形） |
| **序列复杂度** | $O(L^2)$ | $O(L)$ |
| **内存** | $O(L^2)$ | $O(L)$ |
| **长距离依赖** | 直接（一步到位） | 间接（通过多层传播） |
| **可解释性** | 较低（注意力矩阵提供一些线索） | 较高（有几何意义） |
| **当前最佳性能** | 更好 | 略逊（10-15%差距） |
| **实现优化程度** | 高（FlashAttention 等） | 低（还有很大优化空间） |
| **理论分析工具** | 较少 | 丰富（微分几何、代数几何） |

### 13.2 适用场景建议

基于当前的结果，我可以给出一些粗略的适用场景建议：

| 场景 | 推荐架构 | 原因 |
|------|---------|------|
| 短序列 + 关系判断 | GrassmannLM | Plücker 坐标天然适合编码成对关系 |
| 超长序列（>10K tokens） | GrassmannLM（潜力） | 线性复杂度优势明显 |
| 通用语言建模 | Transformer（目前） | 当前性能更优 |
| 需要可解释性的应用 | GrassmannLM | 有几何解释 |
| 受限计算环境 | GrassmannLM（优化后） | 线性复杂度 + 更低内存 |
| 代码生成/数学推理 | Transformer（目前） | 需要复杂的全局依赖 |

## 写在最后

读完这篇论文，我最大的感受是：**深度学习的研究正在从"大力出奇迹"向"巧劲出奇迹"转变。**

以前大家都在堆参数、堆数据、堆算力，看谁能把 Transformer 做得更大。但现在越来越多的人开始思考：有没有更聪明的方法？

Grassmann 流形的方法给了我们一个启示：也许我们不需要给模型一个完全自由的、无结构的巨大空间，而是应该给它一个"有引导"的、有明确结构的较小空间。就像给一个人一张地图和指南针，比给他一个无限大的空旷场地更有效率。

当然，论文也有明显的不足——性能差距、缺乏大规模验证、实现效率低等。但这些都不妨碍它作为一个有价值的研究起点。

在我看来，最有价值的不是 GrassmannLM 这个具体的模型，而是它代表的思维方式：**用结构化的数学对象来设计神经网络。** 这种思维方式在未来可能会催生更多有趣的架构。

最后说一句可能有点偏激的话：如果有一天，深度学习真的变得"可理解、可控、可信赖"，那很可能不是因为找到了更好的训练技巧，而是因为我们找到了更好的数学基础。而 Grassmann 流形这样的工作，就是在铺设这样的基础。

至于"Attention Is All You Need"这句名言，我觉得更准确的版本可能是：**Attention Is Enough, But Not All You Need.**（注意力够用了，但不是你需要的一切。）
