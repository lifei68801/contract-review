# 合同审核技能

对商务合同进行系统化风险审查。

## 审核流程

### Step 1：预审确认

读取合同文本后，先确认以下8项前置信息：

1. 合同类型（产品销售/技术实施/定制开发/订阅协议/混合）
2. 合同金额及与报价单差异
3. 是否涉及第三方分包/渠道分佣
4. 是否存在履约保证金要求
5. 是否涉及定制开发功能
6. 是否涉及源代码交付要求
7. 是否涉及数据安全合规要求
8. 是否涉及第三方知识产权授权

如有任一涉及，提示需提前与运营服务中心报备。

### Step 2：否决条件扫描（7条即时触发）

对合同文本逐一检查以下否决条件，**命中任一条即输出【否决】**：

| # | 检查项 | 检查方式 |
|---|--------|----------|
| 1 | 侵犯我方知识产权（共有IP条款） | 搜索"共同拥有""共同享有知识产权"等表述 |
| 2 | 无违约责任上限 | 搜索"赔偿不封顶""无上限赔偿"，确认是否存在上限条款 |
| 3 | 付款周期超30日 | 提取付款条款中的日期/天数 |
| 4 | 我方须提供所有源代码 | 搜索"源代码""源码交付""全部源码" |
| 5 | 保密期限永久/无期限 | 搜索"永久保密""无期限保密" |
| 6 | 无限连带赔偿责任 | 搜索"无限连带""连带赔偿"（无上限限定） |
| 7 | 数据安全责任无限连带 | 搜索数据安全条款中的连带责任，确认是否有上限 |

### Step 3：七大类红线逐项审查

按以下类别逐条对照审查，详细规则见 [references/redlines.md](references/redlines.md)：

1. **知识产权**（R-001~004）：产品名称/源代码/IP归属/共有IP
2. **违约责任**（R-005~006）：责任上限/直接损失原则
3. **验收&回款**（R-007~012）：退款规则/需求变更/首款/验收/付款周期/背靠背
4. **产品&服务**（R-013~017）：订阅终止/买断维保/升级/扩容/交付周期
5. **保密&数据安全**（R-018~019）：保密期限/数据安全
6. **合规&廉洁**（R-020~021）：反贿赂/税务发票
7. **服务方权益**（R-022~024）：暂停服务/履约金/通知送达

### Step 4：必备条款完整性检查

确认合同是否包含以下必备条款（缺失则为【警告】）：

- 违约责任上限（≤实收费用总额）
- 仅赔偿直接损失（排除间接/惩罚性）
- 首款≥30%（签署后10日内）
- 付款周期≤30日
- 验收环节+支付节点+验收标准附件
- 验收逾期违约金（每天0.5‰，上限3%）
- 暂停服务权（逾期30日）
- 送达条款（含电子送达）

如果是订阅合同，额外检查：
- 订阅自动终止条款
- 扩容费用条款

如果是买断合同，额外检查：
- 维保期限制（1年免费）
- 无"免费升级"承诺

### Step 5：输出审核报告

输出格式：

```
📋 合同审核报告

基本信息：
- 合同类型：xxx
- 合同金额：xxx
- 审核依据：商务合同自审红线标准

🚫 否决项（N/N）：
[如有，列出命中条款+原文+修改建议]

⚠️ 警告项（N/N）：
[如有，列出风险点+对应条款+修改建议]

✅ 必备条款检查：
[逐项列出通过/缺失]

📝 改进建议：
[其他非红线但建议优化的条款]

审核结论：✅ 通过 / ⚠️ 有条件通过 / 🚫 否决
```

## 关键词速查

审核时可使用 [references/keywords.md](references/keywords.md) 快速定位风险条款中的关键词。

## 优先引导原则

- 商务沟通优先引导产品售卖合同
- 招标文件中涉及红线的内容需提前沟通
- 超出红线的条款须在合同审批时提前注明
