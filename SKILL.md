# 通用合同审核技能

为企业提供可定制的商务合同系统化风险审查。

## ⛔ MANDATORY GATE — 初始化检查（不可跳过）

**每次用户要求审核合同时，执行以下检查：**

1. 用 `exec` 检查配置文件是否存在：
   ```bash
   cat ~/.openclaw/workspace/.contract-review-config.json 2>/dev/null || echo "NOT_FOUND"
   ```
2. **如果返回 `NOT_FOUND`** → 立即停止，进入【初始化流程】（见下方），**禁止继续审核**
3. **如果配置存在** → 先确认用户的审核角色（甲方/乙方），再进入审核流程

**🚫 绝对禁止：在配置文件不存在的情况下跳过初始化直接审核合同。**
**🚫 绝对禁止：用户未明确审核角色时自行假设。**

---

## 初始化流程

当配置文件不存在时触发，**必须**按以下步骤执行：

### 1. 确认审核角色

先问用户：**"您是以甲方（采购方）还是乙方（服务/产品提供方）的视角审核合同？"**

- 这决定了所有红线规则的适用方向（如"源代码交付"对甲方和乙方的风险完全不同）
- 角色确认后，写入配置文件的 `review_role` 字段

### 2. 确认企业名称

问用户企业名称，写入 `company` 字段。

### 3. 逐模块引导配置

读取 `references/config-questions.md`，按模块 1-7 依次向用户提问。

- 每次只问一个模块，等用户回答后再问下一个
- 用户回答"按示例"时，使用问题中的示例值
- 用户可随时说"修改合同审核配置"重新设定

### 4. 写入配置文件

所有模块回答完毕后，将配置写入 `~/.openclaw/workspace/.contract-review-config.json`：

```json
{
  "company": "企业名称",
  "review_role": "甲方 或 乙方",
  "version": "1.0.0",
  "created": "YYYY-MM-DD",
  "payment": { ... },
  "liability": { ... },
  "ip": { ... },
  "product_service": { ... },
  "confidentiality": { ... },
  "compliance": { ... },
  "service_rights": { ... }
}
```

配置完成后，**回复用户"配置完成"并立即开始审核合同**（不要等用户再次发送合同文件）。

---

## 审核流程

配置存在时，按以下 5 步执行：

### Step 1：预审确认

读取合同文本后，先确认 8 项前置信息：

1. 合同类型（产品销售/技术实施/定制开发/订阅协议/混合）
2. 合同金额及与报价单差异
3. 是否涉及第三方分包/渠道分佣
4. 是否存在履约保证金要求
5. 是否涉及定制开发功能
6. 是否涉及源代码交付要求
7. 是否涉及数据安全合规要求
8. 是否涉及第三方知识产权授权

如有任一涉及，提示用户需提前评估风险。

### Step 2：否决条件扫描（7条即时触发）

对合同文本逐一检查，**命中任一条即输出【否决】**：

| # | 检查项 | 检查方式 |
|---|--------|----------|
| 1 | 侵犯己方知识产权（共有IP超出配置范围） | 搜索"共同拥有""共同享有知识产权"，对照 `ip` 配置 |
| 2 | 无违约责任上限 | 搜索"赔偿不封顶""无上限赔偿" |
| 3 | 付款周期超过配置最大天数 | 提取付款条款日期，对照 `payment.max_payment_cycle_days` |
| 4 | 己方须提供所有源代码（超出配置范围） | 搜索"源代码""源码交付""全部源码"，对照 `ip` 配置 |
| 5 | 保密期限超过配置上限或永久 | 搜索"永久保密""无期限保密"，对照 `confidentiality` 配置 |
| 6 | 无限连带赔偿责任（无上限） | 搜索"无限连带""连带赔偿" |
| 7 | 数据安全责任无限连带（无上限） | 搜索数据安全条款中的连带责任，对照 `confidentiality` 配置 |

### Step 3：七大类红线逐项审查

按类别逐条对照审查。详细规则见 `references/redlines.md`，数值从配置文件读取：

1. **知识产权** → 对照 `ip`
2. **违约责任** → 对照 `liability`
3. **验收&回款** → 对照 `payment`
4. **产品&服务** → 对照 `product_service`
5. **保密&数据安全** → 对照 `confidentiality`
6. **合规&廉洁** → 对照 `compliance`
7. **服务方权益** → 对照 `service_rights`

### Step 4：必备条款完整性检查

确认合同是否包含以下必备条款（缺失为【警告】）：

- 违约责任上限（≤ 配置比例 × 实收费用）
- 赔偿范围限定为配置范围（排除排除项）
- 首款 ≥ 配置最低比例%（签署后配置期限内）
- 付款周期 ≤ 配置最长天数
- 验收环节 + 支付节点 + 验收标准附件
- 验收逾期违约金（每天配置‰率，上限配置%）
- 暂停服务权（逾期配置天数）
- 送达条款（含电子送达）

订阅合同额外：订阅自动终止条款、扩容费用条款
买断合同额外：维保期限制、无"免费升级"承诺

### Step 5：输出审核报告

```
📋 合同审核报告

基本信息：
- 审核角色：甲方/乙方
- 合同类型：xxx
- 合同金额：xxx
- 审核依据：企业自定义商业底线规则

🚫 否决项（N/N）：
[命中条款+原文+修改建议]

⚠️ 警告项（N/N）：
[风险点+对应条款+修改建议]

✅ 必备条款检查：
[逐项通过/缺失，显示配置值]

📝 改进建议：
[非红线但建议优化的条款]

审核结论：✅ 通过 / ⚠️ 有条件通过 / 🚫 否决
```
