## Context-Compression 技能增强（v3.6.0）

### 问题发现
- AGENTS.md 要求执行 session hooks，但我没遵守
- Real-time memory writing 依赖我主动执行，不是自动的
- hooks 只检查文件存在性，不检测未保存内容

### 解决方案
1. **session-end-hook.sh v2.0**
   - 检测 MEMORY.md 是否有今日更新
   - 检测 daily note 新鲜度
   - 扫描会话文件中的关键词（重要/决定/TODO/偏好）
   - 生成 `.session-alert` 警告文件

2. **mid-session-check.sh** (NEW)
   - 每5分钟扫描当前会话
   - 检测关键词：重要/决定/记住/别忘了/TODO/待办/偏好/我喜欢/我讨厌/千万/务必
   - 输出 JSON 推荐：`SAVE_NOW` 或 `OK`
   - 检查 daily note 新鲜度

### 使用方式
```bash
# 手动检查
~/.openclaw/workspace/skills/context-compression/scripts/mid-session-check.sh

# 添加到 crontab（可选）
*/5 * * * * ~/.openclaw/workspace/skills/context-compression/scripts/mid-session-check.sh >> ~/.openclaw/logs/mid-session.log 2>&1
```

### 记忆机制问题
- **问题**：我没有主动查找上下文，memory 里也没有记录
- **根因**：没有遵守 AGENTS.md 规则 - "重要对话 → 立即写入 memory"
- **改进**：对话发生时立刻写 memory，不等会话结束
