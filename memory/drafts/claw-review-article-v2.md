# 我花一周测了5款Claw产品，结论是：都不如我自己干

一个北师大学生拿"整理就业信息"这个真活儿，把市面上主流的Claw产品全跑了一遍。

结果嘛……挺真实的。

## 先说Claw是个啥

三月中旬，AI圈子里突然炸开一个概念：Claw。

不是龙虾钳子。是一种新产品形态。

通俗点讲，它就是个"AI管家"。你给它接上微信、飞书、Telegram这些聊天工具，它就能在对话框里跟你沟通，跟真人似的。然后你给它装各种"技能"（叫skill），抓网页、管文件、发消息、跑定时任务，理论上什么都能干。

为什么突然火了？2026年初，大厂集体冲进来了。月之暗面出了KimiClaw，腾讯一口气推了WorkBuddy和QClaw两款，MiniMax上了MaxClaw，火山引擎、小米、阿里据说都在排队。"Claw"这个名字怎么来的呢？最早的开源项目叫OpenClaw，logo是只龙虾，claw就是钳子。后来大家习惯性地把这类产品都叫Claw了，就这么简单。

产品多不代表好用。我趁着手头有现成的活儿，把这五款Claw全跑了一遍。有些发现，值得聊聊。

## 测试怎么做的

交代下背景。我在北师大统计学院就业办信息部打工，日常工作是从四个公众号发的招聘信息里，筛出统计学背景能投的岗位，按模板整理好发给学生。每天都要做，重复性高，但需要判断力，不是纯体力活。

就业老师一拍脑袋：你拿这个任务测测那几个Claw呗，看看AI能不能替你干。

指令一模一样：每天17:32，整理前一天17:32到当天17:32之间指定网站新发的招聘信息。招聘和实习分开，按固定模板输出公司名、岗位、地点、投递时间、面试流程、报名链接这些，存成两个txt。来源给了北师大就业网和北京高校毕业生就业网，还告诉它们可以去其他主流招聘网站也找找。

就这么个任务。听起来不难吧？

3月16号到23号，一周，五款产品，从部署到跑任务全程记录。中间踩了不少坑，也发现了一些有意思的细节。下面挨个说。

先报个结论，免得你们看到最后才骂我：五款产品，在"整理就业信息"这个任务上，没有一款能替代真人。最好的那个，也就做到了真人效果的三分之一。最差的，零条。

## OpenClaw：上限高，下限也低

五款里唯一的开源产品，GitHub有源码，社区驱动。也是五款里最折腾的。

部署第一天就给我下马威。Windows得先装WSL，再装Ubuntu子系统，然后在Ubuntu里配Node.js、装OpenClaw。没个扎实的计算机基础和英文底子，搞不定。我最后是靠AI辅助才装好的。让另一个AI教我怎么装这个AI。想想挺魔幻的。

花费倒是不用担心，模型你自己选，API key自己配。穷学生用便宜的，土豪上最贵的，丰俭由人。

坑是真不少。装好之后第一件事，API配置炸了。消息发得出，OpenClaw收得到，就是不回。改配置、换密钥，折腾了好一阵。第二件事更离谱：一开始我发了很多"你好"都没得到回复，日志里堆了一堆无效消息。后来API修好了，它居然没触发"首次聊天"的引导流程（本应该问你是谁、想让它干嘛之类的）。最后是清掉所有日志、重启服务才搞定。

而且每次用之前都得在Ubuntu终端手动跑restart gateway，不然IM连不上。工作目录藏在\\wsl$\Ubuntu-22.04\home\milo\.openclaw下面，涉及到Linux文件系统，不熟的人找起来要命。我一开始也找了好久，后来用AI才定位到的。对普通用户来说，光是找到工作目录这一步就可能卡住半天。

不过OpenClaw有个别的产品没有的能力：你可以在对话里让它自己下载skill。想加什么功能，直接说"帮我装个xxx"，它就去ClawHub上找、下、装。这个过程全自动的，你只要等它装完就行。文件管理也能做，就是容易卡。卡的时候你会看到一个loading状态持续很久，也不知道它是在思考还是在发呆。

定时任务我设好了，3月19号压根没执行。翻日志查，查到3月23号才发现：设了定时任务，还得手动启用。设了等于没设。这个设计让我困惑了很久。

后来手动执行，只保留了1家招聘和1家实习。其他信息它抓到了，筛选的时候太严给过滤掉了。KimiClaw在同一网站上抓到了4条，它只留了1条。抓取能力还行，判断力差点。

还有一点值得一说。OpenClaw的社区在ClawHub上共享skill和plugin，类似一个技能商店。你可以从上面装别人写好的技能包，也可以把自己写的传上去。这个生态虽然是早期，但已经有一批开发者在贡献了。比如有人做了自动总结对话的skill，有人做了定时检查网站更新的skill，还有做PPT生成、论文翻译的。你在其他Claw产品上装skill，大概率也是从ClawHub下的。某种程度上说，OpenClaw定义了这个品类的技能标准。

我的看法：适合有技术背景、愿意折腾的人。上限取决于你装了什么skill、怎么配，下限是你可能被部署过程劝退。想开箱即用？别看这个。

## KimiClaw：最贵，但也最省心（如果它不崩溃的话）

月之暗面的产品，云端服务，充钱即用。不用装任何东西，打开网页就能玩。门槛确实低。

199元/月，五款里最贵。自带Kimi模型，不用自己配key。但也只能用Kimi，想换模型没门。

有专门的KimiClaw用户手册，新手友好。我在飞书上连了它的机器人，然后白屏了。到现在没修好，好在网页版正常。偶尔会请求超时或者页面崩溃，频率嘛……对一个付费产品来说，不够稳。

工作区在云端，网页端能直接编辑配置文件，比OpenClaw那个Linux目录直观多了。Skills我倒没刻意装，感觉它缺什么会自己去找来装。

定时任务的执行方式挺有趣。到点了对话框一点动静都没有，它在后台悄悄干活，干完了才弹出来说"搞定了"。3月19号那次它只摸了北师大就业网，其他网站一律没碰。最后整理出上海浦东发展银行、天津银行、惠州市德赛西威、汤臣倍健四家公司的招聘。实习？一条都没有。

四条。我手动做的话，至少二三十条。

多说一句。KimiClaw还有个小问题：它只能接Kimi模型。这意味着什么呢？你的体验完全取决于Kimi本身的能力。如果Kimi在某方面拉胯，你没办法通过换模型来解决。就好比你买了一台只能加92号油的车，哪怕隔壁95号更便宜更好，你也没辙。对比之下OpenClaw和QClaw都可以自己接模型，灵活度高很多。

不想折腾的话可以试试，部署和使用都简单。但199块配上这个效果和稳定性，性价比我不太想说。

## WorkBuddy：腾讯的笨孩子

腾讯的产品，跟QClaw不是一个团队。公测中，不收费。

本地安装，装完就用。能选几个国内大模型，不多，但有。本地部署的好处是可以操作电脑上的文件，日志也能直接看。

感觉就一个字：笨。

工作目录散在三个地方，没有markdown配置文件，看不到SOUL.md这类自定义的东西。你搞不清它底层逻辑跟OpenClaw一不一样。网页读取经常卡住，我一度怀疑是不是断网了。

定时任务到点了它会疯狂弹工作日志刷屏。跟KimiClaw那个安静干活的样子完全相反。更要命的是：电脑关了它就不干活。周末我电脑没开，周一回来一看，什么都没执行。本地部署的老毛病。

不过抓取方面有个意外：指定的两个网站它一个都没抓到（大概缺相关skill），但在boss直聘上翻出了5家招聘和3家实习。同一批任务，跟KimiClaw找到的公司完全不一样。这倒让我想，也许该把搜索范围扩大一些。

还附带了写代码和办公功能，算是白嫖的额外福利。

能用。但别指望它聪明。免费的东西，薅就完了。

## QClaw：微信直连是真香，但其他一言难尽

腾讯电脑管家团队做的。内测阶段，要兑换码才能免费用。

微信直连。就这四个字，是它最大的卖点。扫码登录之后，你直接在微信上跟它聊天，不用开什么网页或者客户端。一个AI助手住在你微信里，随时发消息让它干活。就冲这个功能，我觉得QClaw如果做好了，推广起来没有对手。

但微信直连之外的部分嘛……

更新太勤了。3月19号部署当天更新了兩三次，上午装的下午就更新，更完上午的对话全丢了。一开始电脑端完全打不开，一开对话就白屏，只能走微信。后来又更了一版才勉强能看。

路径也搞不清楚。我明确说过桌面在D盘，它在C盘给我建了个"桌面"文件夹。让它存到桌面某个文件夹里，存到了C盘。说了两三遍记不住。

Skills安装方便，说一声就自己去装了。Plugin就翻车了。挂梯子也报网络错，本地导入又要求装Node.js，装了又找不到路径。折腾半天没搞定。

安全方面得提一句，有专门的安全沙箱。毕竟电脑管家团队出来的，安全这块是基本功。本地部署+安全沙箱，意味着它既能操作你电脑上的文件，又不会乱跑到不该去的地方。这个设计思路我觉得是对的。

我的感觉是底子不错，腾讯的技术实力摆在那，但产品太嫩了。微信直连如果真能稳定下来，再加上靠谱的能力输出，杀伤力很大。现在嘛，再等等。

我甚至能想象到QClaw成熟之后的样子：微信群里@一下就帮你干活，文件整理、日程安排、信息搜集，全在微信里完成。不用下载任何APP，不用打开任何网页。对中国用户来说，这个体验太顺滑了。所以虽然现在bug一堆，我还是愿意持续关注它。

## MaxClaw：便宜大碗，中规中矩

MiniMax出的，云端部署，跟KimiClaw一样充钱即用。

39元/月。差不多是KimiClaw的五分之一。价格上最有诚意。

用起来比KimiClaw稳定一点。工作区网页端能编辑，Skills导入方便，也有用户手册。定时任务跟Kimi一样，后台悄悄跑，跑完弹出结果。

抓取也只覆盖了北师大就业网。但有意思的是，同样的网站，Kimi整理出4家公司，MaxClaw整理出来七八家（多了兰州财经大学、青岛东软载波、深圳睿联技术、招银网络科技这些）。两个产品结果对不上。要么是网站在不同时段更新了内容，要么两家的模型在信息提取上确实有差异。

缺点是工作区看不了文件，下载也不方便。用起来比较封闭，不像OpenClaw和Kimi那样可以随便折腾工作区。

39块一个月。比Kimi稳，但没Kimi开放。预算有限又不想折腾的话，可以试试。

不过我想吐槽一个点：MaxClaw的工作区居然看不了文件内容。我用KimiClaw的时候，至少能在工作区里直接看到它生成的txt文件，复制粘贴就行。MaxClaw呢，你能看到文件列表，但打不开。想下载？也不太方便。这设计我不理解，一个云端产品，文件管理反而比本地部署的还封闭。希望后续能改。

## Skill和Plugin的生态现状

我还测了一下各产品安装Skill和Plugin的能力。用MaxClaw生成一个简单的Skill和一个Plugin，传到GitHub，让其他Claw去装。

Skill安装五款都过了。毕竟Skill就是一个文件夹加几个配置文件，门槛不高。

Plugin就炸了。MaxClaw自己生成的Plugin，自己都装到了错误目录。KimiClaw更惨，装完直接崩溃。重启、修复、删文件，没用。最后恢复初始设置才救回来。离谱的是，崩溃期间它居然还能自动执行定时任务。崩溃了还在干活，也算一种敬业。反过来说，一个产品崩溃了还能跑定时任务，是不是说明定时任务和主系统之间是隔离的？这也算一个好消息，至少不会因为崩溃丢了任务。

反而是OpenClaw表现最好。安装过程中它发现了Plugin有问题，直接给出了修复建议。我拿着这个建议让MaxClaw改了Plugin代码，再给OpenClaw装，虽然慢，但装上了。这个过程中OpenClaw展现出的错误检测和诊断能力，其他四款都不具备。开源产品能拆开来看内部在干什么，出了问题你能查日志、改配置，封闭产品就不行。这也是为什么我前面说OpenClaw"上限高"的原因。

这说明ClawHub这个技能共享平台虽然在搭，但生态还非常早期。Skill还好说，Plugin涉及更复杂的依赖和权限，各家兼容性参差不齐。你从网上随便下一个Plugin装上去，有概率把系统搞崩。现阶段建议只装验证过的Skill，Plugin别碰。

## 真正的瓶颈在哪

为什么五款产品，没一款能替代真人？

不是单项技术不行。网页抓取能做，信息提取能做，格式化输出也能做。问题出在拼起来不行。

理解力。我的任务要筛"统计学背景可以应聘的岗位"，这个判断很模糊。"数据分析"显然适合统计学投，"市场营销"表面看不像，但如果岗位描述里写了"数据运营、用户行为分析"，其实也可以投。真人会点进去看职责描述再判断，AI只按关键词匹配。"统计""数据""分析"出现就收，没有就跳。该留的漏了，不该留的留下了。

搜索广度。五款产品，要么只抓北师大就业网（KimiClaw、MaxClaw），要么指定网站抓不到（WorkBuddy），要么抓到了但筛选太狠（OpenClaw）。没有一款能做到：你让我查两个网站，我把两个网站查完，还主动去别的主流平台也找找。

这背后有个更深层的问题：Claw产品对"主动"的理解还很弱。我给它的指令是"指定网站+其他主流平台也找找"，但只有WorkBuddy真的去了别的平台（虽然它没抓到指定网站）。其余四款全盯着指定的那一两个网站，没有一个主动拓展搜索范围。你得多明确地告诉它"你去boss直聘也搜一下""你去拉勾也看看"，它才会去。这种被动执行的模式，离真正的"AI助手"还差一截。

可靠性。定时任务该跑的时候不跑（OpenClaw要手动启用，WorkBuddy电脑关了就废），跑了的结果也参差不齐。如果我每天都要检查它做了没有、做得对不对，那检查的成本跟自己干一遍差不多，用它的意义在哪？

单次任务执行，Claw已经能做不少事了。但"持续可靠地完成重复性任务"，这条路还长。

## 选哪个？

技术人员，爱折腾，要控制权：OpenClaw。开源、可定制、模型随便换。愿意研究skill开发的话，它能做到别的产品做不到的事。前提是你得扛得住部署阶段的坑。

普通用户，不想折腾：KimiClaw或MaxClaw。Kimi贵但功能完善，Max便宜但封闭。看预算。

图微信直连的体验：QClaw，但等它再成熟点。现在bug太多。

白嫖党：WorkBuddy和QClaw（能搞到兑换码的话）。免费的，薅就完了。

想拿来做日常工作自动化的：再等等。没有一款能做到稳定可靠地替代人工。当辅助工具用用还行，别当主力。

## 聊点别的

Claw现在的局面，有点像2010年前后的智能手机。iPhone定义了品类，Android阵营一堆厂商冲进来。每家都在做，每家都有毛病。MIUI刷机变砖，HTC电池一天两充，三星死机重启。

Claw大概也在这个阶段。品类定义清楚了：AI Agent + 聊天工具 + 技能系统。但每家的完成度都不够。OpenClaw像早期的Android，能力强门槛高。KimiClaw像iPhone，封闭但易用。QClaw像当年的Windows Phone，有独特卖点（微信直连），但生态跟不上。

2026年3月才到中旬，火山引擎的ArtClaw、小米的MiMoClaw、阿里的JVSclaw据说都已经在做了。再过几个月这个赛道会挤满人。

但热闹不等于好用。

我真正想看到的是，有一款产品先把两件事做到位：定时任务可靠执行、网页信息精准抓取。然后再去卷那些花哨的功能。现在大家都在堆功能、堆生态、堆接入渠道，基础的可靠性反而没人管。

说个具体的例子。我的就业信息整理任务，核心就三个步骤：1. 定时触发；2. 抓取指定网页；3. 按模板输出。三个步骤，每一步在技术上都不是难题。但五款产品，没有一款能把这三步串起来稳定跑通。要么第1步不触发，要么第2步抓不到，要么第3步输出不全。

一个连"每天17:32准时把两个网站的信息整理好"都做不好的产品，你给它装一百个skill又有什么用？
